# 常見選擇題陷阱整理

## 1. 看到「只靠單一措施就能完全防住」通常是陷阱

資安題很常把錯誤選項包裝成很絕對的說法，例如：

* 只要是 OWASP Top 10 就立刻修，不必評估
* 只靠防火牆就能擋 RCE
* 只做字串過濾就能防 Command Injection
* 只靠 WAF 就能擋住所有網頁攻擊
* 只靠微分割就夠，不需要 IAM / 加密 / 監控
* 導入單一產品就能完成 SOC

這類選項的共同問題是：**把複雜問題說成單點解法**。考試很愛把這種話包成「最有效率」「最完整」，但通常是錯的。   

**判斷口訣：**
只要看到「僅靠、完全、足以、唯一、全部都能解決」，先提高警覺。

---

## 2. 看到「絕對化字眼」通常很危險

最常見的錯誤選項特徵就是：

* 一定
* 必然
* 完全
* 只能
* 無需
* 任何狀況都
* 獨立運作、不需搭配其他控制

例如：

* 微分割獨立於其他零信任控制措施運作
* 持續監控不起作用
* WAF 只能防某一種攻擊
* 自動化工具一定能完整找出問題
* TLP:AMBER 任何狀況都不得對客戶散播

這類選項很常因為講得太滿而出錯。   

---

## 3. 前端防護當主防線，通常是陷阱

試卷很愛考這種：

* 用 JavaScript 過濾特殊字元防 Command Injection
* 前端禁 `.gif` 上傳就算防住惡意檔案
* 前端限制欄位格式就代表安全

這類通常錯在：**前端限制可以被繞過**。真正有效的防護一定要落在後端、伺服器端、執行環境或存取控制上。 

**考場判斷：**
看到「前端限制」被描述成核心防護措施，通常不是正解。

---

## 4. 黑名單過濾很常被拿來當錯誤選項

題目很常暗示：

* 移除特殊字元就能防注入
* 過濾私網 IP 字串就能防 SSRF
* 黑名單限制資源名稱就能防資源注入

這類是經典陷阱，因為：

* 可以繞過
* 變形太多
* 不同編碼／協定／語法都可能突破

所以比起黑名單，正確答案通常會偏向：

* allowlist
* 參數化查詢
* 結構化驗證
* 後端授權檢查
* 執行環境隔離。 

---

## 5. 「看到攻擊就立刻做最重處置」不一定對

很多錯誤選項故意把反應寫得很激烈，例如：

* 立即關閉端點電源
* 立刻重灌電腦
* 立刻切斷所有 IT/OT 連線
* 一天內修完所有弱點
* 一出事就先全面阻斷所有流量

但實務題常考的是：**最小影響、有效遏制、兼顧證據保全與業務持續**。
所以正解通常不是最猛的，而是最合適的。

例如：

* EDR 發現 Beacon，較好的作法是隔離網路並保全 RAM / 磁碟證據，不是直接關機。
* OT 場景下要強調「最小影響但高效遏制」，不是粗暴全斷。  

---

## 6. 工具名稱很像，但用途搞錯是高頻陷阱

這科很愛考工具對應錯誤。

常見陷阱：

* **Fortify** 當成網站弱點掃描工具
* **Burp Suite** 當成原始碼分析工具
* **Nmap** 當成 Web 攔截注入平台
* **SonarQube** 當成動態滲透測試主工具
* **防毒 / 沙箱** 當成能解決全部郵件風險的工具

正確思路通常是：

* Fortify → SAST / 原始碼分析
* Burp → Web 攔截、改包、注入測試
* Nessus / Qualys → 弱點掃描
* Nmap → 掃描主機、服務、版本
* Metasploit → 利用與驗證漏洞
* SAST → 適合 CI/CD 提前發現問題。  

**考試技巧：**
先問自己：這題在考「原始碼」、「執行中系統」、「網頁互動」、「主機服務掃描」哪一種，再配工具。

---

## 7. 把「偵測工具」講成「萬用防禦工具」常是錯的

例如：

* IDS 能避免資料外洩
* NDR 最適合阻止未授權資料外傳
* SIEM 本身就能完成所有 SOC 工作
* DNS 反向代理可處理 TCP SYN Flood

這類問題在於：工具定位錯了。
考題很重視你是否知道每種技術是：

* 偏偵測
* 偏阻擋
* 偏回應
* 偏治理
* 偏證據整合

例如資料外洩防護通常是 **DLP**，不是 IDS 或 CDN。 

---

## 8. WAF 題常考「有限有效，不是萬能」

WAF 相關陷阱非常固定：

錯誤選項常寫成：

* WAF 只能防垃圾郵件
* WAF 只能擋 Injection
* WAF 對特徵稍微變化就完全無效
* WAF 能完全阻擋所有網頁攻擊

實務上常被接受的觀念是：

* WAF 有用
* 但可被繞過
* 它是補強，不是唯一防線

所以最穩的答案通常是那種**中間派、務實派**敘述。

---

## 9. 上傳漏洞題，最愛出「只看副檔名」陷阱

常見錯誤選項：

* 只限制 `.gif` `.jpg`
* 只在前端禁傳某副檔名
* 只把檔案搬位置就算安全
* 只看 MIME type 就結束

正確方向通常要同時包含：

* 驗副檔名
* 驗內容
* 轉檔/重編碼
* 獨立上傳目錄
* 不可執行
* 最好放到靜態資源伺服器

也就是說，**單一步驟通常不是正解，組合式防護才像正解**。 

---

## 10. Access Control 題，最常把「表面上有擋」包裝成正確

這類常見陷阱是：

* 頁面顯示「無權限存取」，但實際改另一個參數還是能進去
* 只比對畫面差異，不驗證後端授權
* 認為「有登入」就代表「有授權」

題目很愛考你是否抓到核心：
**真正的授權檢查要在後端，而且要針對資源做驗證。**

所以像這種情況：

* 改 URI / POST 參數就能讀到高權限頁面
  幾乎就是 **Broken Access Control**。

---

## 11. 情境題中，最像正解的通常不是最炫技的，而是最務實的

例如遇到：

* EOS 防火牆
* DDoS
* 勒索事件
* 端點惡意程式
* SOC 建置

錯誤選項很常故意寫成：

* 馬上全面阻斷
* 馬上升級全部設備
* 馬上買新系統
* 馬上做最昂貴措施

但正解往往是：

* 先評估風險
* 採補償控制
* 做最小影響遏制
* 啟動既有應變流程
* 根據場景選最適合、最可行的措施。  

---

## 12. 名詞很像時，考的是你能不能分清邊界

這類很容易失分。

高頻要分清：

* **BCP / DRP / ISCP**
* **SIEM / SOAR / SOC / EDR**
* **SAST / DAST / 滲透測試**
* **DLP / IDS / NDR / CDN**
* **Govern / Identify / Protect / Detect / Respond / Recover**

命題方式常是把功能互相偷換：

* 把 BCP 說成 IT 系統復原
* 把 DRP 說成整體業務持續
* 把 SIEM 說成缺乏關聯分析
* 把 SOAR 說成 Security Design

這類題的關鍵不是背名詞，而是記住「**它到底負責什麼**」。  

---

## 13. 日誌判讀題的陷阱：不要過度解讀

這類題很常設陷阱在「你以為你懂，但其實推論太多」。

例如：

* `cat /etc/passwd` 不是讀明文密碼
* `whoami` 是看當前執行身分，不是看版本
* `phpinfo()` 是蒐集系統資訊
* `find / -perm -u=s -type f` 是找 SUID，不是找 root:root 檔案
* `sleep 5` 常是在驗證是否有命令注入

也就是說，**log 題要回到指令本身的真實作用，不要腦補太遠。**  

---

## 14. 「最新框架版本」題，常考你有沒有注意到新變化

這兩份卷子很明顯有在考新版觀念，例如：

* NIST CSF 2.0 新增 **Govern**
* Zero Trust 不只是網段切割
* 生成式 AI 的安全控制包含 Prompt Injection、DLP、RBAC

陷阱常會放舊說法或不完整說法，讓你選錯。
這種題目的判斷方式是：**新版通常更重治理、風險管理、整合控制，不只技術本身。**   

---

## 15. 多選題最大陷阱：把「看起來也有點對」混進來

多選題最容易錯在：

* 其中 2 個很明顯正確
* 另外 1 個「好像也能接受」
* 但其實不符合題幹範圍、場景或工具定位

例如：

* OT 維運安全題，A/B/C合理，但 D 說不能委外 SOC 就太絕對
* LLM 安全題，Prompt Injection、DLP、RBAC 對；Layer 4 Firewall 不對題
* SecOps 題，SIEM / SOAR / Threat Intelligence 對，但 SSDLC 不一定是該題核心
* 郵件大檔案事件題，加硬碟未必真能防同類事件再次發生

多選題務必檢查：

1. 是否真的對準題幹
2. 是否屬於同一層次的控制
3. 是否只是「有幫助」但不是「此題要的」

# 最後給你一版超短判斷口訣

## 選項長這樣，先懷疑

* 只靠單一措施
* 前端當主防線
* 黑名單萬能
* 最激烈處置一定最好
* 工具用途混淆
* 絕對化字眼
* 把偵測當阻擋
* 把治理框架講成技術細節
* 把 log 功能解讀錯

## 選項長這樣，比較像正解

* 風險導向
* 分層防護
* 後端驗證
* 最小影響遏制
* 補償控制
* 組合式措施
* 工具定位清楚
* 治理 + 技術一起看
