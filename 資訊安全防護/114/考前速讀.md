# 資訊安全防護實務｜考前速讀版

## 1. 最常考主軸

先記這 8 類：

1. **弱點管理與風險評估**
2. **Web 攻擊與修補**
3. **日誌判讀與事件調查**
4. **SOC / SIEM / SOAR / EDR**
5. **NIST / BCP / DRP / ISCP / Zero Trust**
6. **DDoS / 勒索軟體 / 事件應變**
7. **存取控制 / 驗證 / DLP**
8. **DevSecOps / SAST / 工具選用**
    

---

## 2. 必背觀念

### 弱點管理

* 弱點不是「全部立刻修」；要看：

  * 風險等級
  * 實際影響
  * 是否已有 exploit
  * 停機/重開機影響
* 掃描後要有 **複測**
* EOS / EOL 設備要做 **風險評估 + 補償控制**
   

### Web 安全

高頻一定熟：

* SQL Injection
* XSS
* CSRF
* SSRF
* LFI
* 任意檔案上傳
* Command Injection
* Broken Access Control

修補原則：

* SQLi：**Prepared Statement / Parameterized Query**
* XSS：**Output Encoding + CSP**
* CSRF：token / 驗證機制
* Upload：檢查內容、限制副檔名、轉檔、獨立目錄、不可執行
* Command Injection：不能只靠前端過濾
* SSRF：不能只擋私網字串
    

### 存取控制 / 認證

* 改參數讀到別人資料 = **Broken Access Control**
* URL 帶帳密 = 錯
* 密碼明文存放 = 錯
* MFA / RBAC / 最小權限 是常見改善建議
   

---

## 3. 超高頻「哪個最爛／最不適切」陷阱

這類題很常出，看到就小心：

* **只靠前端 JavaScript 過濾**
* **只靠黑名單**
* **只限制副檔名，不驗內容**
* **只靠防火牆擋 RCE**
* **只做字串過濾防 Command Injection**
* **只買單一產品就想做 SOC**
* **把 WAF 當成萬能**
* **看到漏洞就不分輕重全部馬上修**
    

---

## 4. 日誌判讀速記

### Linux / Web 常見惡意意圖

* `phpinfo()`：蒐集系統資訊
* `cat /etc/passwd`：看帳號資訊，不是看明文密碼
* `cat /etc/sudoers`：找可提權帳號
* `cat /proc/self/environ`：看環境變數
* `whoami`：看當前執行身分
* `sleep 5`：常拿來測試是否可注入
* `nc ...`：可能在做反向連線
* `.bashrc` / `crontab`：常見 persistence
    

### 權限維持

常見：

* 寫入 `.bashrc`
* 寫入 `crontab`
* 反向 shell
* SSH 金鑰異常配置


### Windows / AD

* 惡意 PowerShell：常見略過執行政策、背景執行
* Pass the Ticket：AD / Kerberos 題高機率會出

## 5. SOC / SIEM / SOAR / EDR 速記

### SOC

不是產品，是能力中心：

* 持續監控
* 事件回應
* 調查分析
* 報告追蹤

### SIEM

功能：

* 收集 / 匯聚 / 關聯 / 分析日誌
* 即時告警
* 事件查詢與回溯

### SOAR

三關鍵字：

* **Orchestration**
* **Automation**
* **Response**

### EDR

* 端點可提供比純網路流量更有價值的入侵證據
* 發現 Beacon 時：

  * 先隔離網路
  * 同時保全 RAM / 磁碟證據
       

---

## 6. NIST / 治理框架速記

### NIST CSF 2.0

新版最大改變：

* 新增 **Govern**

六大功能：

* Govern
* Identify
* Protect
* Detect
* Respond
* Recover
   

### Govern 重點

* 高層責任
* 治理與決策
* 與整體風險管理整合


### BCP / DRP / ISCP

* **BCP**：整體業務持續
* **DRP**：IT 系統災難復原
* **ISCP**：特定資訊系統的應急準備


### BIA

核心：

* 找出關鍵業務流程
* 看中斷影響
* 排復原優先順序


---

## 7. Zero Trust 速記

* 核心不是「內網就可信」
* 微分割要搭配：

  * IAM
  * 加密
  * 持續監控
* 信任評等常看：

  * 資產狀態
  * 資源內容
  * 環境因素


---

## 8. DDoS / 勒索 / OT 必背

### DDoS

分清楚：

* HTTP Flood
* SYN Flood
* DNS Flood
* ICMP Flood

常見有效緩解：

* Rate Limit
* SYN Cookies
* CDN / 上游清洗
* NGFW / 流量過濾
* 事件時即時溝通


### 勒索軟體

常見行為：

* 刪除/破壞備份
* 影響可用性
* 需要隔離、應變、啟動 BCP

注意：

* `vssadmin`、`wbadmin` 常被拿來對備份下手
   

### OT / ICS

* OT 事件優先級很高，因為可能影響實體安全與營運
* Modbus 異常寫入要靠深度解析規則處理
* IT/OT 分區、DCN、存取限制是重點

## 9. 工具題速記

常考「哪個工具最適合什麼」

* **Fortify**：SAST / 原始碼掃描
* **Burp Suite**：Web 攔截、重送、注入測試
* **Nessus**：系統弱點掃描
* **Nmap**：掃描主機、Port、版本
* **Metasploit**：利用驗證漏洞
* **SonarQube**：偏程式碼品質/靜態分析
* **DAST**：執行中應用測試
* **SAST**：適合整合 CI/CD

## 10. AI / 新題型速記

近年新趨勢要看：

* Prompt Injection

* DLP 規則
* RBAC

## 11. 最後 1 分鐘背誦版

### 看到題目先想

* 這題在考：**攻擊辨識、修補方式、工具選用、治理框架，還是事件應變？**

### 一定要背

* SQLi → Prepared Statement
* XSS → Output Encoding / CSP
* Upload → 驗內容、轉檔、不可執行
* Broken Access Control → 改參數越權
* SOC 不是單一產品
* SIEM = 蒐集/關聯/分析/告警
* SOAR = 編排/自動化/回應
* EDR = 端點證據強
* NIST CSF 2.0 新增 Govern
* BCP / DRP / ISCP 要分清楚
* Zero Trust = 永不預設信任
* WAF 有用，但可被繞過
* 前端過濾不是主要防線
* 黑名單通常不可靠
