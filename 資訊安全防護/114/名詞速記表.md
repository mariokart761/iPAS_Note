# 容易混淆名詞對照速記表

## 1. BCP / DRP / ISCP / BIA

| 名詞       | 中文         | 核心重點                    | 你要怎麼記          |
| -------- | ---------- | ----------------------- | -------------- |
| **BCP**  | 業務持續計畫     | 讓**整體業務**在中斷時仍能維持或恢復運作  | 看的是「公司業務能不能撐住」 |
| **DRP**  | 災難復原計畫     | 讓**IT 系統/基礎設施**在災難後恢復   | 看的是「系統怎麼救回來」   |
| **ISCP** | 資訊系統應急準備計畫 | 針對**特定資訊系統**的應變與復原      | 看的是「某一套系統怎麼應急」 |
| **BIA**  | 業務衝擊分析     | 分析哪些業務最重要、中斷影響多大、復原優先順序 | 看的是「先救哪個業務」    |

**速記口訣：**

* **BCP 看業務**
* **DRP 看 IT**
* **ISCP 看單一系統**
* **BIA 排優先序**
   

---

## 2. SOC / SIEM / SOAR / EDR

| 名詞       | 核心定位          | 容易混淆點      | 正確理解                                          |
| -------- | ------------- | ---------- | --------------------------------------------- |
| **SOC**  | 安全營運中心        | 常被誤認是一套產品  | 它是**人 + 流程 + 技術**的營運能力                        |
| **SIEM** | 蒐集、關聯、分析日誌與告警 | 常被當成整個 SOC | 它是 SOC 的重要平台，不等於 SOC                          |
| **SOAR** | 編排、自動化、回應     | 常被說成是設計工具  | 它重點是**Orchestration / Automation / Response** |
| **EDR**  | 端點偵測與回應       | 常被當成傳統防毒   | 它提供端點層級的偵測、調查、遏制能力                            |

**速記口訣：**

* **SOC = 營運中心**
* **SIEM = 看 log**
* **SOAR = 做自動化應變**
* **EDR = 看端點行為**
     

---

## 3. SAST / DAST / 滲透測試 / 弱點掃描

| 名詞       | 測什麼        | 適合時機      | 常見工具/情境             |
| -------- | ---------- | --------- | ------------------- |
| **SAST** | 原始碼        | 開發中、CI/CD | Fortify、SonarQube 類 |
| **DAST** | 執行中的應用程式   | 系統已跑起來    | Web 動態測試            |
| **滲透測試** | 模擬攻擊者利用弱點  | 上線前後、完整驗證 | Burp、Metasploit 等情境 |
| **弱點掃描** | 找已知弱點/設定問題 | 例行盤點      | Nessus、Qualys 類     |

**速記口訣：**

* **SAST 看程式碼**
* **DAST 看執行中網站**
* **滲測看可不可以打進去**
* **弱掃看已知弱點盤點**
   

---

## 4. SQL Injection / Command Injection / SSRF / LFI

| 名詞                    | 攻擊對象     | 關鍵特徵               | 常見線索                                   |
| --------------------- | -------- | ------------------ | -------------------------------------- |
| **SQL Injection**     | 資料庫查詢    | 注入 SQL 語法          | `' or 1=1 --`、資料庫錯誤                    |
| **Command Injection** | 作業系統指令   | 注入 shell/system 指令 | `; whoami`、`; sleep 5`                 |
| **SSRF**              | 伺服器對外發請求 | 逼伺服器去打別的位址/服務      | `http://127.0.0.1`、`file://`、`dict://` |
| **LFI**               | 本機檔案包含   | 讀取/載入伺服器本地檔案       | `/etc/passwd`、log 檔、設定檔                |

**速記口訣：**

* **SQLi 打資料庫**
* **Command Injection 打作業系統**
* **SSRF 叫伺服器幫你連**
* **LFI 讀本機檔案**
     

---

## 5. Authentication / Authorization / Access Control

| 名詞                 | 中文   | 問的其實是什麼    | 常見錯誤          |
| ------------------ | ---- | ---------- | ------------- |
| **Authentication** | 身分驗證 | 你是不是你      | 密碼錯誤、未登入、憑證問題 |
| **Authorization**  | 授權   | 你能不能做這件事   | 登入了但不該看這頁     |
| **Access Control** | 存取控制 | 系統如何落實權限規則 | 改參數就能看到別人資料   |

**速記口訣：**

* **Authentication = 你是誰**
* **Authorization = 你能做什麼**
* **Access Control = 系統怎麼限制你**
   

---

## 6. Broken Authentication vs Broken Access Control

| 名詞                        | 重點      | 題目常怎麼出                   |
| ------------------------- | ------- | ------------------------ |
| **Broken Authentication** | 驗證機制有問題 | 密碼明文、URL 帶帳密、登入流程不安全     |
| **Broken Access Control** | 權限檢查有問題 | 改 ID、改 POST 參數、低權限讀高權限資料 |

**速記判斷：**

* 問「登入機制、帳密保護」偏 **Authentication**
* 問「越權、改參數、看別人資料」偏 **Access Control**

## 7. IDS / IPS / NDR / DLP / WAF

| 名詞      | 核心用途       | 不要搞混成         |
| ------- | ---------- | ------------- |
| **IDS** | 偵測入侵       | 不一定能主動阻擋      |
| **IPS** | 偵測並阻擋      | 不只是看，不然就變 IDS |
| **NDR** | 網路流量偵測與回應  | 不是專門防資料外洩     |
| **DLP** | 防止敏感資料外流   | 不是一般入侵偵測      |
| **WAF** | 保護 Web 應用層 | 不是萬能，不只防單一攻擊  |

**速記口訣：**

* **IDS 看**
* **IPS 擋**
* **NDR 看網路異常**
* **DLP 管資料外流**
* **WAF 保網站**
   

---

## 8. Zero Trust / Micro-Segmentation / IAM

| 名詞                     | 核心概念        | 容易誤解     |
| ---------------------- | ----------- | -------- |
| **Zero Trust**         | 永不預設信任，持續驗證 | 不是只有切網段  |
| **Micro-Segmentation** | 細粒度網路區隔     | 不是自己單獨就夠 |
| **IAM**                | 身分與存取管理     | 不是只管帳號開通 |

**正確關係：**
Zero Trust 底下會搭配：

* IAM
* 微分割
* 加密
* 持續監控

**速記口訣：**

* **Zero Trust 是理念**
* **Micro-Segmentation 是網路手段**
* **IAM 是身分手段**

## 9. NIST CSF 1.1 vs 2.0

| 版本          | 核心差異                                                   |
| ----------- | ------------------------------------------------------ |
| **CSF 1.1** | 原本五大功能：Identify / Protect / Detect / Respond / Recover |
| **CSF 2.0** | 新增第六大功能：**Govern**                                     |

**Govern 重點：**

* 高層責任
* 策略治理
* 風險管理整合

**速記口訣：**

* **2.0 多一個 Govern**
   

---

## 10. BCP 啟動 vs Incident Response vs Disaster Recovery

| 名詞                    | 何時用                    |
| --------------------- | ---------------------- |
| **Incident Response** | 發現事件當下，先遏制、調查、通報       |
| **BCP**               | 事件影響到核心業務，為了讓業務繼續運作而啟動 |
| **DR**                | 災後要把 IT 系統恢復回來時啟動      |

**速記順序感：**
**先應變、再撐業務、再救系統。**
 

---

## 11. 勒索軟體 vs 一般惡意程式 vs DDoS

| 名詞         | 核心危害              |
| ---------- | ----------------- |
| **勒索軟體**   | 加密檔案、破壞可用性，常伴隨刪備份 |
| **一般惡意程式** | 可能竊資、後門、控制主機，未必加密 |
| **DDoS**   | 用大量流量或請求癱瘓服務      |

**速記口訣：**

* **勒索看加密與可用性**
* **惡意程式看控制與滲透**
* **DDoS 看流量與服務阻斷**

## 12. HTTP Flood / SYN Flood / DNS Flood / ICMP Flood

| 類型             | 打哪一層     | 特徵              |
| -------------- | -------- | --------------- |
| **HTTP Flood** | L7 應用層   | 大量 HTTP 請求壓垮網站  |
| **SYN Flood**  | L4 傳輸層   | 半開連線塞爆資源        |
| **DNS Flood**  | DNS 服務層面 | 大量 DNS 查詢/請求    |
| **ICMP Flood** | 網路層      | 大量 ping/ICMP 封包 |

**速記口訣：**

* **HTTP Flood 打網站**
* **SYN Flood 打連線**
* **DNS Flood 打解析**
* **ICMP Flood 打頻寬/封包處理**


---

## 13. Prompt Injection / Data Leakage / RBAC（AI 安全）

| 名詞                   | 在 AI 場景代表什麼        |
| -------------------- | ------------------ |
| **Prompt Injection** | 用輸入提示操控模型，繞過原本規則   |
| **Data Leakage**     | 模型輸出不該吐出的敏感資訊      |
| **RBAC**             | 限制誰可以問哪些知識、用哪些模型功能 |

**速記口訣：**

* **Prompt Injection 管輸入**
* **DLP/Data Leakage 管輸出**
* **RBAC 管誰可用什麼**


---

## 14. TLP:RED / AMBER / GREEN / CLEAR

| 等級        | 代表意思          |
| --------- | ------------- |
| **RED**   | 只限收件者本人       |
| **AMBER** | 通常限組織內或有需要知道者 |
| **GREEN** | 可在相關社群分享      |
| **CLEAR** | 可公開散播         |

**速記口訣：**

* **RED 最嚴**
* **AMBER 內部/有限分享**
* **GREEN 社群可分享**
* **CLEAR 可公開**


---

## 15. Fortify / Burp Suite / Nessus / Nmap / Metasploit

| 工具             | 你要記的定位         |
| -------------- | -------------- |
| **Fortify**    | 原始碼安全分析，偏 SAST |
| **Burp Suite** | Web 攔截、改包、注入測試 |
| **Nessus**     | 弱點掃描           |
| **Nmap**       | 掃主機、埠、服務版本     |
| **Metasploit** | 利用/驗證漏洞        |

**速記口訣：**

* **Fortify 看碼**
* **Burp 打 Web**
* **Nessus 找洞**
* **Nmap 找服務**
* **Metasploit 驗利用**
