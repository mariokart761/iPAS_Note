# 資安防護名詞速記表｜整合版

## 1. 弱點／威脅／風險／事件

| 名詞 | 意思                   | 重點           |
| -- | -------------------- | ------------ |
| 弱點 | 系統本身存在可被利用的缺陷        | 是「洞」本身       |
| 威脅 | 可能利用弱點的人、行為或惡意程式     | 是會來打洞的東西     |
| 風險 | 弱點被威脅利用後，造成損害的可能性與影響 | 不是只看漏洞本身     |
| 事件 | 已經發生的異常、攻擊或損害        | 不是可能發生，而是已發生 |

**一句話記憶**
洞＝弱點；會利用洞的＝威脅；出事機率與後果＝風險；真的出事＝事件

---

## 2. 弱點掃描／滲透測試／源碼檢測／資安健診／社交工程

| 名詞   | 在做什麼         | 重點                 | 常見陷阱       |
| ---- | ------------ | ------------------ | ---------- |
| 弱點掃描 | 用工具找已知弱點     | 自動化、定期執行、涵蓋面大      | 不等於模擬完整攻擊  |
| 滲透測試 | 模擬駭客實際利用弱點   | 驗證可利用性，可發現部分商業邏輯問題 | 不等於一般弱點掃描  |
| 源碼檢測 | 直接分析程式碼找漏洞   | 偏開發階段              | 不是看系統執行結果  |
| 資安健診 | 針對整體環境做盤點與檢查 | 範圍通常比單一掃描更廣        | 不等於單一工具檢測  |
| 社交工程 | 測試人員是否會被誘騙   | 測的是人，不是系統          | 不是應用系統弱點掃描 |

**一句話記憶**
掃描＝找洞；滲測＝打進去試；源碼＝看程式；健診＝整體檢查；社交工程＝測人

---

## 3. 網路弱點掃描／主機弱點掃描／網站弱點掃描

| 類型     | 在看什麼         | 常見內容                        |
| ------ | ------------ | --------------------------- |
| 網路弱點掃描 | 網路設備與服務暴露面   | 開放埠、服務版本、通訊協定               |
| 主機弱點掃描 | 單一主機的修補與設定狀況 | 帳號、服務、修補、系統設定               |
| 網站弱點掃描 | Web 應用程式漏洞   | SQL Injection、XSS、上傳漏洞、敏感目錄 |

**一句話記憶**
網路掃設備與服務；主機掃作業系統與設定；網站掃 Web 漏洞

## 4. CVE／CVSS

| 名詞   | 意思      | 容易混淆點       |
| ---- | ------- | ----------- |
| CVE  | 漏洞編號    | 是漏洞 ID，不是分數 |
| CVSS | 漏洞嚴重度評分 | 是分數，不是漏洞名稱  |

**一句話記憶**
CVE＝編號；CVSS＝分數

**縮寫全名**

* CVE：Common Vulnerabilities and Exposures
* CVSS：Common Vulnerability Scoring System

---

## 5. SAST／DAST

| 名詞   | 在哪裡測       | 看什麼        | 重點             |
| ---- | ---------- | ---------- | -------------- |
| SAST | 程式執行前／開發階段 | 原始碼        | 靜態分析，不需實際執行程式  |
| DAST | 系統執行中／上線前後 | 執行中的應用程式行為 | 動態測試，要在系統跑起來時測 |

**一句話記憶**
SAST 看原始碼；DAST 看跑起來的系統

**常見陷阱**

* 把 OWASP ZAP 認成 SAST
* 把 SonarQube、Checkmarx、Fortify 認成動態掃描工具

**縮寫全名**

* SAST：Static Application Security Testing
* DAST：Dynamic Application Security Testing
* OWASP：Open Worldwide Application Security Project

---

## 6. SQL Injection／Command Injection／RCE／LFI／SSRF

| 名詞                | 攻擊目標／核心概念     | 典型結果             |
| ----------------- | ------------- | ---------------- |
| SQL Injection     | 注入資料庫查詢語法     | 讀取、竄改、刪除資料       |
| Command Injection | 將惡意命令塞進程式執行流程 | 執行系統命令、控制主機      |
| RCE               | 遠端可執行程式碼或命令   | 攻擊者可遠端接管系統       |
| LFI               | 讀取或載入本機檔案     | 看見機密檔、設定檔、密碼資訊   |
| SSRF              | 逼伺服器代為發送請求    | 存取內網、雲端中介資料、內部服務 |

**一句話記憶**
SQLi＝打資料庫；Command Injection＝打作業系統；RCE＝遠端接管；LFI＝偷看本機檔；SSRF＝叫伺服器幫你連

**常見陷阱**

* 看到 Injection 就亂選
* LFI 不一定直接拿到管理員權限
* RCE 風險通常比 LFI 更高

**縮寫全名**

* RCE：Remote Code Execution
* LFI：Local File Inclusion
* SSRF：Server-Side Request Forgery
* SQL：Structured Query Language

---

## 7. XSS／CSRF

| 名詞   | 本質       | 攻擊對象        | 關鍵概念            |
| ---- | -------- | ----------- | --------------- |
| XSS  | 惡意腳本注入頁面 | 使用者瀏覽器      | 偷 Cookie、執行前端腳本 |
| CSRF | 偽造使用者請求  | 已登入網站的使用者身分 | 借用登入狀態送出請求      |

**一句話記憶**
XSS＝在頁面塞腳本；CSRF＝借你的登入狀態做事

**判斷法**

* 題目提到 script、瀏覽器執行 → 常是 XSS
* 題目提到已登入、幫你送請求 → 常是 CSRF

**縮寫全名**

* XSS：Cross-Site Scripting
* CSRF：Cross-Site Request Forgery

---

## 8. Authentication／Authorization／Access Control

以及 Broken Authentication／Broken Access Control

| 名詞                    | 問的其實是什麼    | 常見情境                     |
| --------------------- | ---------- | ------------------------ |
| Authentication        | 你是不是你      | 密碼錯誤、未登入、憑證問題            |
| Authorization         | 你能不能做這件事   | 登入了但不該看這頁                |
| Access Control        | 系統如何落實權限規則 | 改參數就能看到別人資料              |
| Broken Authentication | 驗證機制有問題    | 密碼明文、URL 帶帳密、登入流程不安全     |
| Broken Access Control | 權限檢查有問題    | 改 ID、改 POST 參數、低權限讀高權限資料 |

**一句話記憶**
Authentication＝你是誰；Authorization＝你能做什麼；Access Control＝系統怎麼限制你

---

## 9. Brute Force／Password Spray

| 名詞             | 做法           | 重點      |
| -------------- | ------------ | ------- |
| Brute Force    | 對單一帳號試很多密碼   | 集中打一個帳號 |
| Password Spray | 用少數常見密碼試很多帳號 | 分散打一堆帳號 |

**一句話記憶**
暴力破解＝一人很多密碼；密碼噴灑＝一個密碼噴很多人

---

## 10. Phishing／社交工程／釣魚郵件／MFA／抗釣魚 MFA

| 名詞       | 意思                | 容易混淆點                  |
| -------- | ----------------- | ---------------------- |
| 社交工程     | 利用人性弱點進行欺騙        | 是大類，不限 email           |
| 釣魚郵件     | 透過 email 誘騙帳密或點連結 | 是社交工程的一種               |
| Phishing | 泛指釣魚行為            | 不只 email，也可能是簡訊、網站、即時通 |
| MFA      | 多因子驗證             | 不代表一定抗釣魚               |
| 抗釣魚 MFA  | 可降低釣魚攻擊成功率的 MFA   | 層級高於一般 MFA             |

**一句話記憶**
社交工程是大傘；釣魚信是其中常見手法；有 MFA 不一定抗釣魚

**縮寫全名**

* MFA：Multi-Factor Authentication
* FIDO：Fast IDentity Online
* PKI：Public Key Infrastructure

---

## 11. SSO／MFA／VPN

| 名詞  | 核心作用           | 容易混淆點     |
| --- | -------------- | --------- |
| SSO | 單一登入，提升便利性     | 不等於強身分驗證  |
| MFA | 多因子驗證，提升身分驗證強度 | 不等於安全通道   |
| VPN | 安全遠端連線通道       | 不等於帳號本身安全 |

**一句話記憶**
SSO 管方便；MFA 管身分；VPN 管通道

**縮寫全名**

* SSO：Single Sign-On
* MFA：Multi-Factor Authentication
* VPN：Virtual Private Network

---

## 12. MITM／DDoS／資料竊取

| 名詞   | 核心差異        | 目標        |
| ---- | ----------- | --------- |
| MITM | 夾在中間偷聽或改資料  | 攔截或竄改通訊   |
| DDoS | 大量流量或請求塞爆服務 | 癱瘓服務、耗盡資源 |
| 資料竊取 | 偷取內容與資訊     | 取得機密資料    |

**一句話記憶**
MITM 是偷聽插話；DDoS 是塞爆癱瘓；資料竊取是偷內容

**縮寫全名**

* MITM：Man-in-the-Middle
* DDoS：Distributed Denial of Service

---

## 13. APT／一般隨機攻擊

| 名詞     | 特性             |
| ------ | -------------- |
| APT    | 目標明確、長期潛伏、隱匿性高 |
| 一般隨機攻擊 | 亂槍打鳥、大量撒網、機會型  |

**一句話記憶**
APT＝有目標、很能躲、待很久；隨機攻擊＝看到誰就打誰

**縮寫全名**

* APT：Advanced Persistent Threat

---

## 14. Zero-day／已知弱點／Virtual Patching／正式修補

| 名詞               | 意思                 | 重點         |
| ---------------- | ------------------ | ---------- |
| Zero-day         | 廠商尚未釋出修補或來不及修補就被利用 | 重點在修補前     |
| 已知弱點             | 已公開且通常已有資訊或修補方式    | 可管理、可追蹤    |
| Virtual Patching | 先用外部控制措施擋風險        | 適合暫時無法立即更新 |
| 正式修補             | 直接把漏洞修掉            | 最根本方式      |

**一句話記憶**
Zero-day＝還來不及補；已知弱點＝知道問題怎麼補；Patch＝治本；Virtual Patch＝先擋住

**縮寫全名**

* Zero-day：Zero-Day Vulnerability / Attack

---

## 15. SOC／SIEM／SOAR／EDR／一般日誌系統

| 名詞     | 核心定位            | 容易混淆點            |
| ------ | --------------- | ---------------- |
| SOC    | 安全營運中心          | 不是單一產品，而是人＋流程＋技術 |
| SIEM   | 收集、關聯分析、告警、協助應變 | 不等於整個 SOC        |
| SOAR   | 編排、自動化、回應       | 重點是自動化流程         |
| EDR    | 端點偵測與回應         | 不等於傳統防毒          |
| 一般日誌系統 | 收集、保存紀錄         | 不一定有關聯分析與應變能力    |

**一句話記憶**
SOC 是中心；SIEM 看 log；SOAR 做自動化應變；EDR 盯端點；一般 log 系統只是存資料

**補充**
SIEM＝SIM＋SEM

**縮寫全名**

* SOC：Security Operations Center
* SIEM：Security Information and Event Management
* SOAR：Security Orchestration, Automation, and Response
* EDR：Endpoint Detection and Response
* SIM：Security Information Management
* SEM：Security Event Management

---

## 16. Event Viewer／Syslog／Facility／Severity／Priority

| 名詞           | 意思               | 重點                               |
| ------------ | ---------------- | -------------------------------- |
| Event Viewer | Windows 內建事件檢視工具 | 看登入、系統、應用程式事件                    |
| Syslog       | 通用日誌標準／機制        | 常見於 Linux 與網通設備                  |
| Facility     | 事件來源類別           | 例如 mail、daemon                   |
| Severity     | 嚴重程度             | 例如 Debug、Warning、Alert、Emergency |
| Priority     | 綜合值              | 由 Facility 與 Severity 組合而成       |

**一句話記憶**
Windows 看 Event Viewer；Linux/設備常看 Syslog；Facility 看誰發的；Severity 看多嚴重；Priority 是綜合值

**縮寫全名**

* Syslog：System Logging Protocol / System Log

---

## 17. Debug／Warning／Alert／Emergency

| 等級        | 意思          |
| --------- | ----------- |
| Debug     | 最詳細，除錯用     |
| Warning   | 有異常但未到最嚴重   |
| Alert     | 很嚴重，需立即處理   |
| Emergency | 最高級，系統不可用等級 |

**一句話記憶**
越往 Emergency 越嚴重；Debug 是最細不是最嚴重

## 18. Delivery／Exploitation／Installation（Cyber Kill Chain）

| 階段           | 意思            | 常見例子      |
| ------------ | ------------- | --------- |
| Delivery     | 把攻擊送到目標手上     | 惡意附件、惡意連結 |
| Exploitation | 開始利用漏洞／觸發惡意行為 | 點開檔案、觸發漏洞 |
| Installation | 惡意程式落地安裝      | 木馬植入系統    |

**一句話記憶**
Delivery＝送到你面前；Exploitation＝開始中招；Installation＝惡意程式住進去

**高頻陷阱**
郵件附件、URL 連結屬於 Delivery，不是 Installation

---

## 19. CIA：可用性／機密性／完整性

| 原則  | 重點      | 例子        |
| --- | ------- | --------- |
| 可用性 | 能不能正常使用 | 勒索加密、DDoS |
| 機密性 | 有沒有被偷看  | 資料外洩      |
| 完整性 | 有沒有被改掉  | 資料遭竄改     |

**一句話記憶**
不能用→可用性；被偷看→機密性；被改掉→完整性

**高頻陷阱**
勒索軟體主要考可用性，不是先選機密性

**縮寫全名**

* CIA：Confidentiality, Integrity, Availability

---

## 20. IT／OT／ICS 與 CIA 優先順序

| 場景       | 常見優先順序          |
| -------- | --------------- |
| IT       | 機密性 → 完整性 → 可用性 |
| OT / ICS | 可用性 → 完整性 → 機密性 |

**一句話記憶**
IT 怕洩密；OT 怕停機

**縮寫全名**

* IT：Information Technology
* OT：Operational Technology
* ICS：Industrial Control Systems
* CIA：Confidentiality, Integrity, Availability

---

## 21. 備份／備援／復原／RTO／3-2-1 備份原則／離線備份

| 名詞       | 在做什麼                 | 重點              |
| -------- | -------------------- | --------------- |
| 備份       | 把資料複製保存              | 保留資料            |
| 備援       | 準備替代系統或環境            | 留替身             |
| 復原       | 事故後讓服務恢復             | 救回來             |
| RTO      | 可接受的最大恢復時間           | 問多久恢復營運，不是多久備份完 |
| 3-2-1 原則 | 3 份資料、2 種媒體、1 份異地或離線 | 不只是多存幾份         |
| 離線備份     | 不直接連線的備份             | 較不易被勒索一起加密      |

**一句話記憶**
備份＝留資料；備援＝留替身；復原＝救回來；RTO＝多久恢復；3-2-1 重點在不同媒體＋異地／離線；防勒索最有感的是離線備份

**常見陷阱**

* 有備份不代表可快速復原
* 備份也可能被污染
* 不是多存幾份就叫 3-2-1

**縮寫全名**

* RTO：Recovery Time Objective

## 22. BCP／DRP／ISCP／BIA

| 名詞   | 中文         | 核心重點                |
| ---- | ---------- | ------------------- |
| BCP  | 業務持續計畫     | 讓整體業務在中斷時仍能維持或恢復運作  |
| DRP  | 災難復原計畫     | 讓 IT 系統／基礎設施在災難後恢復  |
| ISCP | 資訊系統應急準備計畫 | 針對特定資訊系統的應變與復原      |
| BIA  | 業務衝擊分析     | 分析哪些業務最重要、影響多大、優先順序 |

**一句話記憶**
BCP 看業務；DRP 看 IT；ISCP 看單一系統；BIA 排優先序

**縮寫全名**

* BCP：Business Continuity Plan
* DRP：Disaster Recovery Plan
* ISCP：Information System Contingency Plan
* BIA：Business Impact Analysis

## 23. Incident Response／BCP 啟動／Disaster Recovery

| 名詞                | 何時用               |
| ----------------- | ----------------- |
| Incident Response | 發現事件當下，先遏制、調查、通報  |
| BCP               | 事件影響核心業務時，讓業務繼續運作 |
| Disaster Recovery | 災後要把 IT 系統恢復回來時啟動 |

**一句話記憶**
先應變，再撐業務，再救系統

## 25. Identify／Protect／Detect／Respond／Recover／Govern（NIST CSF）

| 功能       | 重點                | 常見例子        |
| -------- | ----------------- | ----------- |
| Identify | 資產、風險、治理、盤點       | 資產管理、風險評估   |
| Protect  | 存取控制、訓練、資料保護、技術保護 | MFA、加密、權限控管 |
| Detect   | 發現異常              | 日誌、監控、告警    |
| Respond  | 事件來了先處理           | 隔離、調查、抑制    |
| Recover  | 恢復營運              | 還原、復原程序     |
| Govern   | 高層責任、策略治理、風險整合    | 2.0 新增重點    |

**一句話記憶**
先盤點，再保護，再偵測，出事就應變，最後復原；CSF 2.0 多一個 Govern

**高頻陷阱**
Asset Management、Risk Assessment 比較偏 Identify，不是 Protect

**縮寫全名**

* NIST：National Institute of Standards and Technology
* CSF：Cybersecurity Framework
* MFA：Multi-Factor Authentication

## 27. Defense in Depth／Least Privilege／Zero Trust／Micro-Segmentation／IAM

| 名詞                 | 核心概念        | 容易誤解           |
| ------------------ | ----------- | -------------- |
| Defense in Depth   | 多層防護        | 不是只裝一套防毒或一台防火牆 |
| Least Privilege    | 只給必要權限      | 偏帳號與操作面的控制原則   |
| Zero Trust         | 永不預設信任，持續驗證 | 不是只有切網段        |
| Micro-Segmentation | 細粒度網路區隔     | 是網路手段，不是全部     |
| IAM                | 身分與存取管理     | 不只是帳號開通        |

**一句話記憶**
縱深防禦＝多層；最小權限＝少給；Zero Trust 是理念；Micro-Segmentation 是網路手段；IAM 是身分手段

**縮寫全名**

* IAM：Identity and Access Management

## 28. IDS／IPS／NDR／DLP／WAF

| 名詞  | 核心用途       | 不要搞混成         |
| --- | ---------- | ------------- |
| IDS | 偵測入侵       | 不一定能主動阻擋      |
| IPS | 偵測並阻擋      | 不只是看，不然就變 IDS |
| NDR | 網路流量偵測與回應  | 不是專門防資料外洩     |
| DLP | 防止敏感資料外流   | 不是一般入侵偵測      |
| WAF | 保護 Web 應用層 | 不是萬能，也不只防單一攻擊 |

**一句話記憶**
IDS 看；IPS 擋；NDR 看網路異常；DLP 管資料外流；WAF 保網站

**縮寫全名**

* IDS：Intrusion Detection System
* IPS：Intrusion Prevention System
* NDR：Network Detection and Response
* DLP：Data Loss Prevention
* WAF：Web Application Firewall

## 31. HSTS／HTTPS／SSLStrip

| 名詞       | 意思              | 容易混淆點             |

HTTPS 是有加密；HSTS 是強制不能降級

## 34. Hash／Encryption／對稱式加密／非對稱式加密／數位簽章

| 名詞         | 特性        | 用途           |
| ---------- | --------- | ------------ |
| Hash       | 不可逆       | 常用於密碼儲存、摘要比對 |
| Encryption | 可逆，要有金鑰解密 | 保護資料內容       |
| 對稱式加密      | 同一把金鑰加解密  | 適合大量資料       |
| 非對稱式加密     | 公私鑰配對     | 常用於金鑰交換      |
| 數位簽章       | 驗證來源與完整性  | 不是拿來加密大量資料   |

**一句話記憶**
Hash 算摘要，理論上不能還原；Encryption 可解回來；大量資料用對稱；交換金鑰用非對稱；驗證真偽用簽章

## 35. 密碼歷程／最短使用效期／最小長度

| 名詞     | 功能          |
| ------ | ----------- |

| 最短使用效期 | 防止立刻改回舊密碼   |

## 36. 勒索軟體／一般惡意程式／Webshell／後門

| 名詞       | 核心危害              |

## 37. HTTP Flood／SYN Flood／DNS Flood／ICMP Flood

| 類型         | 打哪一層     | 特徵              |
| ---------- | -------- | --------------- |
| HTTP Flood | L7 應用層   | 大量 HTTP 請求壓垮網站  |
| SYN Flood  | L4 傳輸層   | 半開連線塞爆資源        |
| DNS Flood  | DNS 服務層面 | 大量 DNS 查詢或請求    |
| ICMP Flood | 網路層      | 大量 ping/ICMP 封包 |

**一句話記憶**
HTTP Flood 打網站；SYN Flood 打連線；DNS Flood 打解析；ICMP Flood 打頻寬或封包處理

**縮寫全名**

* HTTP：Hypertext Transfer Protocol
* SYN：Synchronize
* DNS：Domain Name System
* ICMP：Internet Control Message Protocol
* L4：Layer 4
* L7：Layer 7

# 考前超短版總背

* 弱點＝洞；威脅＝會利用洞的；風險＝出事可能與影響；事件＝真的出事
* 弱點掃描＝找已知洞；滲透測試＝模擬攻擊；源碼檢測＝看碼；社交工程＝測人
* CVE＝編號；CVSS＝分數
* SAST＝看原始碼；DAST＝看跑起來的系統
* SQLi＝打資料庫；Command Injection＝打 OS；SSRF＝叫伺服器代打；LFI＝讀本機檔；RCE＝遠端接管
* XSS＝塞腳本；CSRF＝借登入狀態送請求
* Authentication＝你是誰；Authorization＝你能做什麼
* Brute Force＝一帳號很多密碼；Password Spray＝一密碼很多帳號
* SSO＝方便；MFA＝強驗證；VPN＝安全通道
* MITM＝攔截；DDoS＝塞爆
* APT＝有目標、潛伏久
* Zero-day＝還沒補就被打；Virtual Patch＝先擋；Patch＝治本
* SOC＝中心；SIEM＝看 log；SOAR＝自動化應變；EDR＝看端點
* CIA：不能用＝可用性；被偷看＝機密性；被改掉＝完整性
* 3-2-1＝3 份、2 媒體、1 異地或離線
* RTO＝多久恢復，不是多久備份完
* BCP＝業務；DRP＝系統；ISCP＝單系統；BIA＝排優先
* OWASP＝Web 風險；MITRE ATT&CK＝攻擊技術；NIST CSF＝治理；SP 800-61＝事故應變
* Zero Trust 是理念；Micro-Segmentation 是手段；IAM 是身分管理
* IDS 看；IPS 擋；DLP 防外洩；WAF 保網站
* VLAN 是方法；隔離是目的
* ping 通不代表服務通
* Hash 不可逆；Encryption 可逆
* STIX 管內容；TAXII 管傳送
* TLP：RED 最嚴，CLEAR 可公開
* 紅打、藍守、紫協調
