# 資訊安全防護實務｜考前速讀版

## 1. 必背核心觀念

### 弱點 vs 威脅 vs 事件

* **弱點（Vulnerability）**：系統、設計、設定、控制上的缺陷，可被利用
* **威脅（Threat）**：可能利用弱點造成損害的來源或行為
* **資安事件（Incident）**：已實際發生、影響系統或資料安全的狀況

### 資安三要素

* **CIA**

  * **Confidentiality 機密性**
  * **Integrity 完整性**
  * **Availability 可用性**

### 常考框架

* **NIST CSF 五大功能**

  * Identify
  * Protect
  * Detect
  * Respond
  * Recover
* **MITRE ATT&CK**

  * 常考：Initial Access、Credential Access、Lateral Movement
* **OWASP Top 10**

  * 常考：Injection、Security Misconfiguration、Logging and Monitoring Failures、Broken Access Control
* **OWASP CDM**

  * 橫軸：NIST CSF 五功能
  * 縱軸：Devices / Applications / Networks / Data / Users

---

## 2. 弱點掃描與工具用途

### 掃描類型

* **網路弱點掃描**：找開放埠、服務、版本、網路設備風險
* **主機弱點掃描**：檢查主機設定、修補、服務、系統暴露面
* **網站弱點掃描**：SQL Injection、XSS、檔案上傳、敏感目錄等
* **源碼掃描**：檢查程式碼中的安全缺陷

### 工具對照

* **Nessus / OpenVAS**：弱點掃描
* **Acunetix / OWASP ZAP**：Web 弱點掃描
* **Checkmarx / Fortify**：源碼檢測
* **Nmap**：主機、埠、服務探測，不是純弱點掃描主工具
* **Wireshark**：封包分析
* **Sqlmap**：SQL Injection 檢測與利用
* **Metasploit**：漏洞利用與測試
* **Burp Suite / ZAP**：Web 測試
* **Aircrack-ng / Kismet**：無線網路分析
* **John the Ripper / Hashcat**：密碼破解

### 弱點處理流程

1. 找出弱點
2. 查 **CVE / CVSS**
3. 修補更新
4. 若暫時不能修補，用補償控制
   例如：

   * 防火牆阻擋
   * 關閉服務/連接埠
   * 虛擬修補
   * 網段隔離

---

## 3. 常見攻擊手法

### Web 攻擊

* **SQL Injection**：注入惡意 SQL，竄改/讀取資料
* **XSS**：把惡意腳本塞進網頁，讓使用者瀏覽器執行
* **CSRF**：誘使已登入使用者送出非預期請求
* **RCE**：遠端執行程式碼
* **Webshell**：在網站主機植入後門腳本

### 帳號攻擊

* **Brute Force**：暴力破解，針對單一帳號多密碼
* **Password Spray**：少數常見密碼，去試很多帳號
* **Credential Access**：竊取帳密、雜湊、Token、票證等

### 其他常見

* **MITM**：中間人攻擊，攔截或改寫通訊
* **Phishing**：釣魚，騙取帳密或誘導點擊惡意內容
* **APT**：持續性、針對性、高隱匿
* **DDoS**：分散式阻斷服務
* **DLL Side Loading**：利用合法程式載入惡意 DLL
* **Zero-day**：廠商尚未正式修補前就被利用的漏洞

---

## 4. 帳號安全與 MFA

### 密碼政策

* 最小長度越長越好
* 密碼歷程：避免重複使用舊密碼
* 最短使用效期：避免立刻改回舊密碼
* 離職帳號要及時停用

### MFA

* 高頻重點：**抗網釣 MFA**
* 常見較佳：

  * **FIDO**
  * **PKI-based MFA**
* 考題常提醒：

  * 單純加長密碼 ≠ MFA
  * 有 MFA 不代表一定抗釣魚

### 最小權限

* 使用者只給工作必需權限
* 例外請求先做：

  * 風險確認
  * 補償控制

## 5. 日誌、監控、SOC

### Windows 常考

* **Event Viewer**：看登入、登出、系統事件
* **4625**：登入失敗

### Syslog（RFC5424）

* 常考：

  * **Facility**
  * **Severity**
  * **Priority**
* **Debug**：最詳細，適合除錯
* Priority 計算觀念要熟

### Log 管理重點

* 時間同步很重要
* 最佳成本效益常見做法：

  * 主要 NTP 伺服器對外校時
  * 其他設備都向內部 NTP 校時
* 原始稽核紀錄不可任意刪除

### SOC

* **事前**：情資蒐集、預防控制
* **事中**：事件監看、告警、分析
* **事後**：根因分析、抑制擴散、改善

---

## 6. 事件應變必背

### 正確處置

* 保留原始紀錄
* 確認根因
* 修補漏洞
* 隔離受影響設備
* 需要時可委外專業廠商

### 常見錯誤

* 刪除原始 log
* 事件未控管就延後處理
* 發現異常仍讓可疑設備持續連線

### 勒索軟體事件常看哪些 log

* 防火牆 / IDS / IPS / 其他網路設備
* 主機作業系統事件
* 防毒 / EDR 告警

---

## 7. 網路防護與系統強化

### Defense in Depth（縱深防禦）

* 不是只做單點防護
* 要在 **主機、網路、應用程式、帳號、資料** 多層防護

### 常見硬化

* 關閉不必要服務與埠
* 阻擋高風險埠對外
* 定期更新修補
* 不共用管理者密碼
* 停用不安全協定或舊版服務

### 高頻埠

* **3389**：RDP
* **445**：SMB
* 公網面若非必要應嚴格限制

### 遠端維護安全

* 使用 **VPN**
* 限制來源位置
* 啟用 **MFA**
* 控管廠商設備安全
* 遠端維護不是靠 SSO 就能解決全部問題

---

## 8. Web / API 安全

### 常考觀念

* 檔案上傳要限制格式，不可只靠防毒
* 缺少 **HSTS** 可能遭受：

  * MITM
  * SSLStrip
* API 保護常見改善：

  * 驗證 Token 簽章
  * 檢查 Token 時效
  * 回應資料去機敏
  * 加強安全日誌

### Logging

* 安全日誌應記錄：

  * IP
  * 時間
  * 帳號
  * 事件類型
  * 成功/失敗
* 不應把敏感日誌放在可公開存取位置

---

## 9. 滲透測試、紅藍紫隊

### 滲透測試

* 是授權下的安全測試
* 不可未經同意就做
* 目的是模擬攻擊驗證弱點
* 不應影響正常營運與資料完整性

### PTES 大致流程

* 前期溝通
* 情報收集
* 漏洞分析
* 利用/測試
* 報告

### 紅藍紫隊

* **紅隊**：模擬攻擊者
* **藍隊**：偵測、防禦、應變
* **紫隊**：促進紅藍合作，不是只有書面演練

---

## 10. AD / 內網攻擊常識

### 常見方向

* LDAP 列舉
* DNS Zone Transfer
* Password Spray
* 取得 AD 控制權
* 擷取 **ntds.dit**

### ntds.dit 提取常見方式

* ntdsutil
* vssadmin
* diskshadow

---

## 11. OT / ICS / Purdue Model

### OT 環境考點

* IT 與 OT 防護邏輯不同
* 工控環境通常更重視：

  * **可用性**
  * **完整性**
  * 再來才是機密性

### 常見保護措施

* 網段分區
* 防火牆政策
* 廠商遠端維護走 VPN
* 虛擬修補保護無法更新設備
* 連線裝置管理與區隔

---

## 12. 備份與加密

### 3-2-1 原則

* **3** 份資料
* **2** 種不同媒體
* **1** 份異地

### 安全通訊

* 大量資料傳輸：**對稱式加密**
* 金鑰交換：**非對稱式加密**

---

# 考前最容易混淆的點

## 這幾個最常被拿來出陷阱

* **弱點掃描 ≠ 滲透測試**
* **Nmap ≠ 完整弱點掃描工具**
* **SSO ≠ 遠端維護安全的核心措施**
* **有 MFA ≠ 一定抗釣魚**
* **只裝防毒 ≠ 足夠防檔案上傳風險**
* **刪除 log ≠ 正確事件處置**
* **紫隊 ≠ 純書面演練**
* **OT 安全優先順序不一定是 CIA 的一般 IT 順序**

# 最後的衝刺建議

這科最常考的不是純背誦，而是：

1. **看到情境能判斷攻擊類型**
2. **知道該看哪種 log**
3. **知道哪種防護措施最立即、最合理**
4. **知道哪個工具做哪件事**
5. **知道哪個選項是“最不合適”**

所以考前請優先熟：

* 弱點掃描/工具
* 事件應變
* 日誌/Syslog/Windows Event
* MFA/帳號安全
* 網路防護與埠控管
