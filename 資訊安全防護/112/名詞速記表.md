# 最容易混淆名詞對照表

## 1. 弱點掃描 vs 滲透測試 vs 源碼檢測 vs 社交工程

| 名詞   | 在做什麼       | 重點                | 常見陷阱         |
| ---- | ---------- | ----------------- | ------------ |
| 弱點掃描 | 用工具找已知弱點   | 自動化、定期執行、涵蓋面大     | 不是模擬完整攻擊     |
| 滲透測試 | 模擬駭客實際利用弱點 | 驗證可利用性，可找部分商業邏輯問題 | 不等於一般弱點掃描    |
| 源碼檢測 | 直接分析程式碼找漏洞 | 屬於 SAST，偏開發階段     | 不是看系統執行結果    |
| 社交工程 | 測試人員是否會被誘騙 | 測「人」不是測系統         | 不是應用系統弱點偵測工具 |

**一句話記憶**

* 弱點掃描：找洞
* 滲透測試：打進去試試看
* 源碼檢測：看程式碼
* 社交工程：騙人上當

  

---

## 2. SAST vs DAST

| 名詞   | 全名         | 在哪裡測       | 看什麼        |
| ---- | ---------- | ---------- | ---------- |
| SAST | 靜態應用程式安全測試 | 程式執行前/開發階段 | 原始碼        |
| DAST | 動態應用程式安全測試 | 系統執行中/上線前後 | 執行中的應用程式行為 |

**一句話記憶**

* SAST：看原始碼
* DAST：看跑起來的系統

**常見陷阱**

* 把 OWASP ZAP 認成 SAST
* 把 SonarQube、Checkmarx、Fortify 認成動態掃描工具

 

---

## 3. SQL Injection vs Command Injection

| 名詞                | 攻擊目標   | 影響          | 修補方向         |
| ----------------- | ------ | ----------- | ------------ |
| SQL Injection     | 資料庫查詢  | 讀取、竄改、刪除資料  | 參數化查詢、輸入驗證   |
| Command Injection | 作業系統命令 | 執行系統命令、控制主機 | 白名單、避免直接拼接命令 |

**一句話記憶**

* SQLi：打資料庫
* Command Injection：打作業系統

**常見陷阱**

* 看到 Injection 就亂選
* 以為都一樣靠過濾特殊字元就能解決

 

---

## 4. RCE vs Command Injection vs LFI

| 名詞                | 核心概念          | 典型結果           |
| ----------------- | ------------- | -------------- |
| RCE               | 遠端可執行程式碼/命令   | 攻擊者能遠端控制系統     |
| Command Injection | 將惡意命令塞進程式執行流程 | 執行系統命令         |
| LFI               | 讀取本機檔案        | 看到機密檔、設定檔、密碼資訊 |

**一句話記憶**

* RCE：遠端接管
* Command Injection：塞命令進去跑
* LFI：偷看本地檔案

**常見陷阱**

* LFI 不一定直接拿到管理員權限
* RCE 風險通常比 LFI 更高



---

## 5. XSS vs CSRF

| 名詞   | 本質       | 攻擊對象        | 關鍵概念            |
| ---- | -------- | ----------- | --------------- |
| XSS  | 惡意腳本注入頁面 | 使用者瀏覽器      | 偷 Cookie、執行前端腳本 |
| CSRF | 偽造使用者請求  | 已登入網站的使用者身分 | 借用使用者身分送出請求     |

**一句話記憶**

* XSS：在頁面塞腳本
* CSRF：借你的登入狀態做事

**判斷法**

* 題目提到 script、瀏覽器執行 → 常是 XSS
* 題目提到已登入、幫你送請求 → 常是 CSRF



---

## 6. 風險接受 vs 補償控制 vs 修補

| 名詞   | 意思          | 什麼情況用   |
| ---- | ----------- | ------- |
| 修補   | 直接把漏洞修掉     | 最根本     |
| 補償控制 | 暫時用其他方式降低風險 | 無法立即修補時 |
| 風險接受 | 經評估後接受殘餘風險  | 要有審查與核准 |

**一句話記憶**

* 修補：治本
* 補償控制：先擋著
* 風險接受：知道有風險但決定承擔

**常見陷阱**

* 「先不修，以後再說」不等於正當風險接受
* 補償控制也不等於永遠不用修



---

## 7. 零時差攻擊 vs 已知弱點

| 名詞            | 差別             |
| ------------- | -------------- |
| 零時差（Zero-day） | 廠商尚未釋出修補或來不及修補 |
| 已知弱點          | 已有漏洞資訊與修補方式    |

**一句話記憶**

* Zero-day：沒補丁或補丁來不及
* 已知弱點：知道問題，也通常知道怎麼補

**常見陷阱**

* 零時差不是完全無法防
* WAF、IPS、虛擬修補可以降低風險，但不是根治



---

## 8. APT vs 一般隨機攻擊

| 名詞     | 特性             |
| ------ | -------------- |
| APT    | 目標明確、長期潛伏、隱匿性高 |
| 一般隨機攻擊 | 亂槍打鳥、大量撒網      |

**一句話記憶**

* APT：有目標、很能躲、待很久
* 隨機攻擊：看到誰就打誰



---

## 9. 社交工程 vs 釣魚郵件

| 名詞   | 關係           |
| ---- | ------------ |
| 社交工程 | 大類，利用人性弱點    |
| 釣魚郵件 | 社交工程最常見的一種手法 |

**一句話記憶**

* 社交工程是大傘
* 釣魚信是傘下最常見那一支



---

## 10. Delivery vs Exploitation vs Installation（Cyber Kill Chain）

| 階段           | 意思            | 常見例子      |
| ------------ | ------------- | --------- |
| Delivery     | 把攻擊送到目標手上     | 惡意附件、惡意連結 |
| Exploitation | 開始利用漏洞/觸發惡意行為 | 點開檔案、觸發漏洞 |
| Installation | 惡意程式落地安裝      | 木馬植入系統    |

**一句話記憶**

* Delivery：送到你面前
* Exploitation：開始中招
* Installation：惡意程式住進去

**高頻陷阱**

* 郵件附件、URL 連結是 **Delivery**，不是 Installation

 

---

## 11. 可用性 vs 機密性 vs 完整性

| 原則                  | 重點      | 例子        |
| ------------------- | ------- | --------- |
| 可用性 Availability    | 能不能正常使用 | 勒索加密、DDoS |
| 機密性 Confidentiality | 有沒有被偷看  | 資料外洩      |
| 完整性 Integrity       | 有沒有被改掉  | 資料遭竄改     |

**一句話記憶**

* 不能用 → 可用性
* 被偷看 → 機密性
* 被改掉 → 完整性

**高頻陷阱**

* 勒索軟體主要考 **可用性**，不是先選機密性

 

---

## 12. 備份 vs 備援 vs 復原

| 名詞 | 在做什麼         |
| -- | ------------ |
| 備份 | 把資料複製保存      |
| 備援 | 準備替代系統/環境可接手 |
| 復原 | 發生事故後讓服務恢復   |

**一句話記憶**

* 備份：留資料
* 備援：留替身
* 復原：救回來

**常見陷阱**

* 有備份不代表可快速復原
* 備份也可能被污染



---

## 13. RTO vs 備份時間

| 名詞   | 意思         |
| ---- | ---------- |
| RTO  | 可接受的最大恢復時間 |
| 備份時間 | 備份作業跑多久    |

**一句話記憶**

* RTO 問的是「多久恢復營運」
* 不是「多久備份完」



---

## 14. 3-2-1 備份原則 vs 一般多份存放

| 名詞       | 重點                   |
| -------- | -------------------- |
| 3-2-1 原則 | 3 份資料、2 種媒體、1 份異地/離線 |
| 一般多份存放   | 只是多留幾份，不一定安全         |

**一句話記憶**

* 不是多存幾份就叫 3-2-1
* 關鍵是 **不同媒體 + 異地/離線**



---

## 15. SIEM vs 一般日誌系統

| 名詞     | 功能                    |
| ------ | --------------------- |
| 一般日誌系統 | 收集、保存紀錄               |
| SIEM   | 收集 + 關聯分析 + 告警 + 協助應變 |

**一句話記憶**

* Log：存資料
* SIEM：拿來看威脅、分析事件

**補充**

* SIEM = SIM + SEM



---

## 16. Hash vs Encryption

| 名詞            | 特性          |
| ------------- | ----------- |
| Hash 雜湊       | 不可逆、常用於密碼儲存 |
| Encryption 加密 | 可逆，要有金鑰解密   |

**一句話記憶**

* Hash：算出摘要，理論上不能還原
* Encryption：加密後可解回來

**高頻陷阱**

* 密碼儲存通常講雜湊，不是明文，也不是單純可逆加密



---

## 17. 密碼歷程 vs 最短使用效期 vs 最小長度

| 名詞     | 功能          |
| ------ | ----------- |
| 密碼歷程   | 防止重複使用舊密碼   |
| 最短使用效期 | 防止立刻改回舊密碼   |
| 最小長度   | 增加密碼組合與破解難度 |

**一句話記憶**

* 歷程：不能重複
* 最短效期：不能馬上改回
* 最小長度：增加難猜程度



---

## 18. OWASP Top 10 vs MITRE ATT&CK

| 名詞           | 在看什麼         |
| ------------ | ------------ |
| OWASP Top 10 | Web 應用常見風險類別 |
| MITRE ATT&CK | 攻擊者戰術、技術、程序  |

**一句話記憶**

* OWASP：Web 弱點分類表
* ATT&CK：駭客怎麼打的知識庫

 

---

## 19. NIST CSF vs NIST SP 800-61

| 名詞             | 用途       |
| -------------- | -------- |
| NIST CSF       | 整體資安治理框架 |
| NIST SP 800-61 | 資安事故處理指引 |

**一句話記憶**

* CSF：管整體資安
* 800-61：管事故應變

 

---

## 20. Identify vs Protect（NIST CSF）

| 功能       | 重點                |
| -------- | ----------------- |
| Identify | 資產、風險、治理、盤點       |
| Protect  | 存取控制、訓練、資料保護、技術保護 |

**一句話記憶**

* Identify：先知道你有什麼、風險在哪
* Protect：再開始保護它

**高頻陷阱**

* Asset Management、Risk Assessment 比較偏 Identify，不是 Protect



---

## 21. ISO 27001 vs 純技術防護

| 名詞        | 核心                      |
| --------- | ----------------------- |
| ISO 27001 | 資訊安全管理系統，重制度、風險、稽核、持續改善 |
| 純技術防護     | 防火牆、防毒、EDR、掃描工具等        |

**一句話記憶**

* ISO 27001 不是單一技術
* 它是整套管理系統



---

## 22. 網路隔離 vs VLAN 切割

| 名詞      | 差別            |
| ------- | ------------- |
| 網路隔離    | 降低不同區域互通與橫向移動 |
| VLAN 切割 | 一種實作分段的方法     |

**一句話記憶**

* VLAN 是工具
* 網路隔離是目的

 

---

## 23. ping 通 vs 服務可用

| 名詞     | 差別              |
| ------ | --------------- |
| ping 通 | ICMP 有回應，基本網路可達 |
| 服務可用   | 指定協定與埠號真的有通     |

**一句話記憶**

* ping 通不代表 syslog、web、mail 一定通

**高頻考點**

* syslog 常見：UDP 514



---

## 24. 離線備份 vs 一般備份

| 名詞   | 差別               |
| ---- | ---------------- |
| 一般備份 | 可能仍連在線上環境        |
| 離線備份 | 不直接連線，較不易被勒索一起加密 |

**一句話記憶**

* 防勒索最有感的是離線備份，不只是有備份而已



---

## 25. DDoS vs 資料竊取

| 名詞   | 核心差異      |
| ---- | --------- |
| DDoS | 癱瘓服務、耗盡資源 |
| 資料竊取 | 偷內容、偷資訊   |

**一句話記憶**

* DDoS 打的是可用性，不是以偷資料為主
