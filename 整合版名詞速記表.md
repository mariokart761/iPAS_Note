# 名詞速記表｜整合版

## 1. 識別 vs 驗證 vs 授權 vs 稽核

| 名詞 | 在問什麼   | 白話意思      | 常見陷阱           |
| -- | ------ | --------- | -------------- |
| 識別 | 你是誰    | 提出身分      | 只有說自己是誰，不代表已證明 |
| 驗證 | 你真的是你嗎 | 查驗身分真偽    | 有帳號不等於通過驗證     |
| 授權 | 你可以做什麼 | 決定可存取哪些資源 | 驗證通過不代表什麼都能做   |
| 稽核 | 你做過什麼  | 留下可追查紀錄   | 稽核不是授權，也不是驗證   |

**秒記**

* 識別：報名字
* 驗證：看證件
* 授權：決定能進哪個房間
* 稽核：監視器和紀錄簿

---

## 2. 帳號管理 vs 身分驗證 vs 授權

| 名詞   | 重點              | 例子            |
| ---- | --------------- | ------------- |
| 帳號管理 | 帳號建立、停用、盤點、期限管理 | 久未登入帳號、臨時帳號停用 |
| 身分驗證 | 確認你是誰           | 密碼、MFA、OTP    |
| 授權   | 確認你可以做什麼        | 角色權限、最小權限     |

**速記一句**
帳號管理管生命週期，驗證管你是誰，授權管你能做什麼。

**縮寫全名**

* MFA = Multi-Factor Authentication
* OTP = One-Time Password

---

## 3. MFA vs 2FA vs OTP vs FIDO2

| 名詞    | 核心意思         | 速記            |
| ----- | ------------ | ------------- |
| MFA   | 多因子驗證        | 多種因子          |
| 2FA   | 雙因子驗證        | 兩種因子          |
| OTP   | 一次性密碼        | 常作為 MFA 的一種方式 |
| FIDO2 | 無密碼 / 強式驗證標準 | 現代化身分驗證       |

**關係**

* 2FA 是 MFA 的一種。
* OTP 是工具，常用於 MFA / 2FA。
* FIDO2 是較現代化的驗證標準。

**縮寫全名**

* MFA = Multi-Factor Authentication
* 2FA = Two-Factor Authentication
* OTP = One-Time Password
* FIDO2 = Fast Identity Online 2

---

## 4. IAM vs 存取控制 vs RBAC vs SoD

| 名詞   | 重點        | 白話意思         |
| ---- | --------- | ------------ |
| IAM  | 身分與權限整體管理 | 管誰、怎麼登入、能做什麼 |
| 存取控制 | 控制資源使用    | 誰能看、誰能改、誰能執行 |
| RBAC | 角色式授權     | 用角色分權限       |
| SoD  | 職務區隔      | 關鍵工作不要同一人全包  |

**秒記**

* IAM：大傘
* 存取控制：核心功能
* RBAC：常見授權方法
* SoD：避免內控與舞弊風險

**縮寫全名**

* IAM = Identity and Access Management
* RBAC = Role-Based Access Control
* SoD = Segregation of Duties

---

## 5. Need-to-Know vs Least Privilege

| 名詞              | 核心意思          | 速記       |
| --------------- | ------------- | -------- |
| Need-to-Know    | 只知道工作需要知道的資訊  | 不該知道就別知道 |
| Least Privilege | 只給完成工作最少需要的權限 | 不該有的權限不給 |

**差異**

* Need-to-Know 偏資訊知道多少。
* Least Privilege 偏系統能做多少。

**縮寫全名**

* Need-to-Know = 僅知原則
* Least Privilege = 最小權限原則

---

## 6. 最小權限 vs 最大特權

| 名詞   | 意思           | 哪個正確 |
| ---- | ------------ | ---- |
| 最小權限 | 只給完成工作所需最少權限 | 正確   |
| 最大特權 | 給最多最好管理      | 通常錯  |

**常見陷阱**
零信任、IAM、存取控制題很常用「最大特權」當干擾選項。

---

## 7. 關注方 vs 內外部議題 vs 法遵要求

| 名詞    | 是什麼               | 例子              |
| ----- | ----------------- | --------------- |
| 關注方   | 對組織有要求或受影響的人 / 單位 | 客戶、主管機關、供應商     |
| 內外部議題 | 會影響管理系統的環境因素      | 組織結構、產業環境、跨國營運  |
| 法遵要求  | 必須遵守的法令或契約要求      | 個資法、GDPR、合約資安條款 |

**速記一句**
關注方是「誰」，議題是「環境」，法遵是「規則」。

**縮寫全名**

* GDPR = General Data Protection Regulation

---

## 8. 建立全景 vs 風險辨識

| 名詞   | 重點    | 常放進去的內容                  |
| ---- | ----- | ------------------------ |
| 建立全景 | 先理解背景 | 法規、內外部議題、利害關係人、業務情境、技術環境 |
| 風險辨識 | 再找風險  | 哪些資產、弱點、威脅、事件可能造成風險      |

**常見陷阱**
看到法規要求、主管機關函文、客戶合約，常比較接近建立全景，而不是直接進入風險處理。

---

## 9. 風險辨識 vs 風險分析 vs 風險評估 vs 風險處理

| 名詞   | 核心動作           | 白話意思        | 速記           |
| ---- | -------------- | ----------- | ------------ |
| 風險辨識 | 找出風險           | 有哪些風險       | 找風險          |
| 風險分析 | 分析可能性與衝擊       | 風險有多大       | 看多大          |
| 風險評估 | 依準則判斷優先順序與可接受性 | 哪些先處理、能不能接受 | 排優先 / 判斷要不要管 |
| 風險處理 | 決定並執行作法        | 降低、避免、移轉、接受 | 採行動          |

**速記句**
先找 → 再算 → 再排 → 再做。

---

## 10. 風險接受準則 vs 風險胃納

| 名詞     | 白話意思       | 差別           |
| ------ | ---------- | ------------ |
| 風險接受準則 | 多大風險算可接受   | 比較像判斷門檻      |
| 風險胃納   | 組織願意承受多少風險 | 比較像高層容忍方向與程度 |

**秒記**

* 胃納：大方向能吃多辣
* 接受準則：這道菜辣度幾分以下可接受

**縮寫全名**

* Risk Appetite = 風險胃納

---

## 11. 固有風險 vs 殘餘風險

| 名詞   | 白話意思       | 何時看   |
| ---- | ---------- | ----- |
| 固有風險 | 還沒做控制前的風險  | 初始評估  |
| 殘餘風險 | 做完控制後剩下的風險 | 處理後再看 |

**常見陷阱**

* 控制後結果不能再叫固有風險。
* 題目若說已做控制措施，通常要想到殘餘風險。

---

## 12. 風險擁有者 vs 資產擁有者 vs 使用者

| 名詞    | 核心意思              | 速記     |
| ----- | ----------------- | ------ |
| 風險擁有者 | 有權決定風險怎麼處理、是否接受的人 | 決定風險的人 |
| 資產擁有者 | 對資產負責的人           | 負責資產的人 |
| 使用者   | 使用系統或資產的人         | 用的人    |

**考點**

* 不是使用者決定風險處理。
* 常見正解：風險擁有者核可風險處理與殘餘風險。

---

## 13. 風險避免 vs 風險降低 / 緩解 vs 風險轉移 / 分擔 vs 風險接受 / 保留

| 名詞        | 白話意思      | 例子                 |
| --------- | --------- | ------------------ |
| 風險避免      | 不做了       | 結束某業務 / 不自行做高風險活動  |
| 風險降低 / 緩解 | 加控制措施     | 加 MFA、WAF、EDR、權限控管 |
| 風險轉移 / 分擔 | 分給別人承擔一部分 | 保險、委外、合約分攤         |
| 風險接受 / 保留 | 知道有風險但接受  | 成本過高，接受可接受風險       |

**常見陷阱**

* 加保險不是降低，是偏轉移 / 分擔。
* 結束業務不是降低，是避免。

**縮寫全名**

* MFA = Multi-Factor Authentication
* WAF = Web Application Firewall
* EDR = Endpoint Detection and Response

---

## 14. 預防控制 vs 偵測控制 vs 矯正控制

| 類型   | 核心意思 | 例子            | 速記  |
| ---- | ---- | ------------- | --- |
| 預防控制 | 事前阻止 | MFA、密碼政策、教育訓練 | 先擋住 |
| 偵測控制 | 發現異常 | IDS、日誌檢查、監控   | 抓到它 |
| 矯正控制 | 事後恢復 | 備份還原、補救措施     | 救回來 |

**超短記法**

* 預防：不讓它發生
* 偵測：知道它發生了
* 矯正：發生後修回來

**縮寫全名**

* MFA = Multi-Factor Authentication
* IDS = Intrusion Detection System

---

## 15. 資產 vs 弱點 vs 威脅 vs 事件

| 名詞 | 白話意思      | 例子          |
| -- | --------- | ----------- |
| 資產 | 有價值要保護的東西 | 系統、資料、人員、服務 |
| 弱點 | 可被利用的缺口   | 未修補漏洞、弱密碼   |
| 威脅 | 可能利用弱點的來源 | 駭客、惡意員工、天災  |
| 事件 | 實際發生的事情   | 帳號被盜、資料外洩   |

**秒記**

* 資產：被保護
* 弱點：洞
* 威脅：會打這個洞的人 / 事
* 事件：真的發生了

---

## 16. 機密性 vs 完整性 vs 可用性

| 名詞  | 在保護什麼    | 常見例子       |
| --- | -------- | ---------- |
| 機密性 | 不該看的人看不到 | 加密、權限控制    |
| 完整性 | 資料不能被亂改  | 雜湊、簽章、變更控管 |
| 可用性 | 要用時能用    | 備援、備份、負載平衡 |

**秒判法**

* 外洩 → 機密性
* 被改 → 完整性
* 不能用 → 可用性

**縮寫全名**

* CIA = Confidentiality, Integrity, Availability

---

## 17. 備份 vs 備援

| 名詞 | 白話意思       | 常考方向        |
| -- | ---------- | ----------- |
| 備份 | 把資料另存一份    | 復原、RPO、資料恢復 |
| 備援 | 服務不中斷的替代設施 | 可用性、HA、RTO  |

**秒記**

* 備份：找回資料
* 備援：服務不中斷

**縮寫全名**

* RPO = Recovery Point Objective
* HA = High Availability
* RTO = Recovery Time Objective

---

## 18. RTO vs RPO vs MTD vs BIA

| 名詞  | 問的是什麼    | 白話意思         |
| --- | -------- | ------------ |
| RTO | 多久要恢復    | 系統停多久內要回來    |
| RPO | 允許失去多少資料 | 最多能丟多少時間內的資料 |
| MTD | 最長可中斷多久  | 超過就撐不住       |
| BIA | 業務衝擊分析   | 評估哪些系統最重要    |

**秒記**

* RTO：恢復時間
* RPO：資料損失容忍
* MTD：最大可忍中斷
* BIA：先判斷重要性

**縮寫全名**

* RTO = Recovery Time Objective
* RPO = Recovery Point Objective
* MTD = Maximum Tolerable Downtime
* BIA = Business Impact Analysis

---

## 19. ALE vs ARO vs SLE

| 名詞  | 意思            | 速記   |
| --- | ------------- | ---- |
| SLE | 單一事件發生一次會損失多少 | 一次損失 |
| ARO | 一年大概發生幾次      | 年發生率 |
| ALE | 一年總預期損失       | 年損失  |

**公式**
ALE = SLE × ARO

**縮寫全名**

* ALE = Annualized Loss Expectancy
* ARO = Annualized Rate of Occurrence
* SLE = Single Loss Expectancy

---

## 20. ISMS vs PIMS

| 名詞   | 全名                                     | 重點                      | 速記  |
| ---- | -------------------------------------- | ----------------------- | --- |
| ISMS | Information Security Management System | 資訊安全管理系統，核心是保護資訊資產與風險管理 | 管資安 |
| PIMS | Privacy Information Management System  | 個資保護管理系統，偏個人資料治理        | 管個資 |

**速記一句**
27001 管資安，27701 管個資。

| -------------- | ----------- | ---- |
| ISO 27001:2013 | 舊版          | 舊    |
| ISO 27001:2022 | 新版，導入時優先認這版 | 新、主考 |

現在考導入，先想到 2022。

## 22. ISO 27001 vs ISO 27002 vs ISO 27005 vs ISO 27014

| ISO 27005 | 資安風險管理       | 不是治理總框架    |
| ISO 27014 | 資安治理         | 不是隱私標準     |

**秒記**

* 27001：制度主體
* 27002：控制指南
* 27005：風險
* 27014：治理

**縮寫全名**

* ISMS = Information Security Management System

| ISO 27011 | 電信產業資安     | 電信場景    |

* 27701 / BS 10012：隱私個資
* 62443：工控
* 27011：電信

* OT = Operational Technology

## 25. 個資法 vs 資通安全管理法 vs GDPR

| 名詞      | 管什麼              | 常見情境          |
| ------- | ---------------- | ------------- |
| 個人資料保護法 | 個人資料蒐集、處理、利用與保護  | 會員資料、告知義務、外洩  |
| 資通安全管理法 | 公務機關與特定非公務機關資安要求 | 機關責任等級、通報應變   |
| GDPR    | 歐盟個資保護法規         | 歐盟居民資料、跨境資料處理 |

**速記一句**
個資法管個資，資安法管機關資安，GDPR 管歐盟個資。

**縮寫全名**

* GDPR = General Data Protection Regulation

---

## 26. 個資最小蒐集 vs 目的外利用

| 名詞    | 意思         | 常見陷阱    |
| ----- | ---------- | ------- |
| 最小蒐集  | 只蒐必要資料     | 正確      |
| 目的外利用 | 拿去做原本沒說的用途 | 常違反個資原則 |

**常見錯誤選項特徵**

* 盡可能蒐集全部資料作行銷
* 先收再說，以後可能有用

## 27. 告知義務 vs 放棄權利

| 名詞   | 核心意思             | 速記    |
| ---- | ---------------- | ----- |
| 告知義務 | 告知蒐集目的、對象、方式、期間等 | 要先說清楚 |
| 放棄權利 | 要求當事人放棄法定權利      | 通常違法  |

**考點**
可以告知用途，但不能要求放棄法定權利。

| SOC 3 | 適合公開給一般大眾     | 公開版  |

SOC 3 = 公開看。

## 29. MAC vs ACL vs RBAC

| 名詞   | 全名                        | 核心概念         | 速記    |
| ---- | ------------------------- | ------------ | ----- |
| MAC  | Mandatory Access Control  | 由系統或政策強制決定權限 | 系統硬規定 |
| ACL  | Access Control List       | 對某資源列出誰能做什麼  | 資源掛名單 |
| RBAC | Role-Based Access Control | 依角色授權        | 看角色   |

**速記一句**
MAC 看政策，ACL 看名單，RBAC 看角色。

## 30. 主體 Subject vs 資源 Resource

| 名詞          | 白話意思      | 零信任裡怎麼看     |
| ----------- | --------- | ----------- |
| 主體 Subject  | 發起動作的人或程序 | 使用者、服務、裝置代理 |
| 資源 Resource | 被存取的東西    | 資料、系統、應用、服務 |

**常見陷阱**
很多題目會把資料、設備、運算服務都說成主體；其實很多時候它們更偏資源。

## 31. PEP vs PDP vs PE vs PA

| 名詞  | 角色    | 白話意思           |
| --- | ----- | -------------- |
| PDP | 政策決策點 | 做出准不准的總體決策位置   |
| PEP | 政策落實點 | 真正執行攔或放的地方     |
| PE  | 政策引擎  | 判斷該不該放行        |
| PA  | 政策管理者 | 接到判斷後安排執行、建立連線 |

**秒記**

* PE 負責想
* PA 負責安排
* PDP 是決策區
* PEP 是執行口

**縮寫全名**

* PDP = Policy Decision Point
* PEP = Policy Enforcement Point
* PE = Policy Engine
* PA = Policy Administrator

---

## 32. 零信任 vs 傳統邊界防護 vs 縱深防禦

| 名詞     | 核心意思          | 速記    |
| ------ | ------------- | ----- |
| 傳統邊界防護 | 比較相信內網，重點守邊界  | 城牆式防守 |
| 零信任    | 內外都不預設信任，持續驗證 | 誰都先不信 |
| 縱深防禦   | 多層防護、層層阻擋     | 多層防線  |

**考場關鍵**

* 內網不等於安全。
* VPN 不等於零信任。
* 零信任重每次驗；縱深防禦重多層擋。

**縮寫全名**

* VPN = Virtual Private Network

---

## 33. VPN vs ZTA vs DMZ vs 網段隔離

| 名詞   | 用途              | 速記       |
| ---- | --------------- | -------- |
| VPN  | 建立安全遠端連線 / 加密通道 | 安全通道     |
| ZTA  | 對每次存取持續驗證與授權    | 每次都要重新確認 |
| DMZ  | 對外服務區，與內網隔離     | 外網緩衝區    |
| 網段隔離 | 降低橫向移動與部門互通     | 內部分艙     |

**差異**

* VPN 解決的是通道保護。
* ZTA 解決的是信任模型與存取決策。
* DMZ 是對外區。
* 網段隔離是內部切開。

**縮寫全名**

* VPN = Virtual Private Network
* ZTA = Zero Trust Architecture
* DMZ = Demilitarized Zone

---

## 34. Firewall vs IDS vs IPS vs WAF vs EDR

| 名詞       | 主要用途      | 最常見誤解        |
| -------- | --------- | ------------ |
| Firewall | 網路邊界流量控管  | 不等於 Web 專用防護 |
| IDS      | 偵測可疑行為    | 偏告警，不一定主動阻擋  |
| IPS      | 偵測並阻擋     | 比 IDS 多了阻擋能力 |
| WAF      | 保護 Web 應用 | 不是萬用防護       |
| EDR      | 端點偵測與回應   | 不是邊界設備       |

**秒記**

* 遠端接入：VPN
* Web 防護：WAF
* 邊界管制：Firewall
* 只看不擋：IDS
* 看了會擋：IPS
* 主機端偵防：EDR

**縮寫全名**

* IDS = Intrusion Detection System
* IPS = Intrusion Prevention System
* WAF = Web Application Firewall
* EDR = Endpoint Detection and Response

---

## 35. 防毒軟體 vs EDR

| 名詞      | 特性           | 差異          |
| ------- | ------------ | ----------- |
| 防毒軟體 AV | 基本惡意程式防護     | 偏簽章、已知威脅    |
| EDR     | 偵測、調查、回應端點威脅 | 更強調行為、調查、反應 |

**常見陷阱**

* 防毒 ≠ EDR
* EDR 通常能力更完整

**縮寫全名**

* AV = Antivirus
* EDR = Endpoint Detection and Response

## 38. SBOM vs 原始碼檢測 vs 滲透測試 vs 弱點掃描

| 名詞    | 核心意思           | 速記     |
| ----- | -------------- | ------ |
| SBOM  | 列出用了哪些軟體元件     | 成分表    |
| 原始碼檢測 | 檢查程式碼安全問題      | 看程式碼   |
| 弱點掃描  | 自動找已知弱點        | 自動找洞   |
| 滲透測試  | 模擬攻擊驗證弱點是否可被利用 | 真的打進去試 |

**差異**

* SBOM 不是弱掃工具，而是幫你知道用了哪些第三方元件。
* 弱掃偏廣、偏自動。
* 滲透測試偏深、偏驗證攻擊可行性。

**縮寫全名**

* SBOM = Software Bill of Materials

## 40. 委外監督 vs 技術移轉 vs 合約控管

| ---- | ----------- | -------------- |
| 委外監督 | 持續看受託者做得好不好 | 稽核、追蹤、驗證       |
| 技術移轉 | 把知識或能力交給委託方 | 教育訓練、文件交付      |
| 合約控管 | 事前把要求寫清楚    | 驗收標準、安全需求、責任分工 |

**常見陷阱**

## 41. 日誌保存 vs 事件通報 vs 事件處置 vs 稽核軌跡

| 名詞   | 重點             | 速記   |
| ---- | -------------- | ---- |
| 日誌保存 | 保留跡證，供追查分析     | 留證據  |
| 事件通報 | 依規定向主管機關或相關方通報 | 報出去  |
| 事件處置 | 控制擴散、補救、復原     | 處理現場 |

# 最後濃縮版｜必背口訣

1. **識別是誰，驗證真假，授權能做什麼，稽核留紀錄。**
2. **辨識列風險，分析算大小，評估排優先，處理採行動。**
3. **建立全景先看背景，風險辨識才開始找風險。**
4. **固有風險在前，殘餘風險在後。**
5. **避免是不做，降低是加控，轉移是分攤，接受是承擔。**
6. **IAM 是大傘，RBAC 是方法，SoD 是分工。**
7. **最小權限永遠比最大特權更像正解。**
8. **VPN 連遠端，WAF 擋 Web，EDR 看端點。**
9. **備份救資料，備援保不中斷。**
10. **RTO 看恢復時間，RPO 看可丟資料量。**
11. **27001 制度、27002 控制、27005 風險、27014 治理。**
12. **零信任裡，主體是來存取的，資源是被存取的。**
13. **OTP 是工具，MFA 是做法，FIDO2 是現代標準。**
14. **防火牆擋流量，IDS/IPS 抓入侵，WAF 顧網站。**
15. **EDR 看行為，防毒偏擋已知惡意程式。**
16. **DDoS 打服務，Credential Stuffing 打帳號。**
17. **SBOM 是成分表，不是弱掃工具。**
18. **ZTA 管信任模型，VPN 管加密通道。**
19. **ISRA 看資安，BIA 看中斷，DPIA 看個資。**
20. **27001 管資安，27701 管個資，42001 管 AI。**
