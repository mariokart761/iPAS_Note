# 容易混淆名詞對照速記表

## 1. ISMS vs PIMS

| 名詞   | 全名                                     | 重點                      | 速記      |
| ---- | -------------------------------------- | ----------------------- | ------- |
| ISMS | Information Security Management System | 資訊安全管理系統，核心是保護資訊資產與風險管理 | **管資安** |
| PIMS | Privacy Information Management System  | 個資保護管理系統，偏個人資料治理        | **管個資** |

### 延伸

* **ISO 27001**：ISMS 主標準
* **ISO 27701**：PIMS 延伸標準，通常建立在 27001 之上。

**速記一句：**
27001 管資安，27701 管個資。

---

## 2. ISO 27001:2022 vs ISO 27001:2013

| 名詞             | 重點          | 速記       |
| -------------- | ----------- | -------- |
| ISO 27001:2013 | 舊版          | 舊        |
| ISO 27001:2022 | 新版，導入時優先認這版 | **新、主考** |

113-2 直接考到導入制度依循應選 **ISO/IEC 27001:2022**。

**速記一句：**
現在考導入，先想到 **2022**。

---

## 3. 關注方 vs 內外部議題 vs 法遵要求

| 名詞    | 是什麼             | 例子              |
| ----- | --------------- | --------------- |
| 關注方   | 對組織有要求或受影響的人/單位 | 客戶、主管機關、供應商     |
| 內外部議題 | 會影響 ISMS 的環境因素  | 組織結構、產業環境、跨國營運  |
| 法遵要求  | 必須遵守的法令或契約要求    | 個資法、GDPR、合約資安條款 |

試題常把這三個混在一起考，但它們不是同一層次。 

**速記一句：**
關注方是「誰」，議題是「環境」，法遵是「規則」。

---

## 4. 個資法 vs 資通安全管理法 vs GDPR

| 名詞      | 管什麼              | 常見情境          |
| ------- | ---------------- | ------------- |
| 個人資料保護法 | 個人資料蒐集、處理、利用與保護  | 會員資料、告知義務、外洩  |
| 資通安全管理法 | 公務機關與特定非公務機關資安要求 | 機關責任等級、通報應變   |
| GDPR    | 歐盟個資保護法規         | 歐盟居民資料、跨境資料處理 |

  

**速記一句：**
個資法管個資，資安法管機關資安，GDPR 管歐盟個資。

---

## 5. SOC 1 vs SOC 2 vs SOC 3

| 名詞    | 重點            | 速記      |
| ----- | ------------- | ------- |
| SOC 1 | 財務報導控制相關      | 財報      |
| SOC 2 | 安全、可用性、機密性等控制 | 控制細節    |
| SOC 3 | 適合公開給一般大眾     | **公開版** |

113-1 直接考「最適合發布給一般大眾閱讀」的是 **SOC 3**。

**速記一句：**
SOC 3 = 公開看。

---

## 6. MAC vs ACL vs RBAC

| 名詞   | 全名                        | 核心概念         | 速記        |
| ---- | ------------------------- | ------------ | --------- |
| MAC  | Mandatory Access Control  | 由系統或政策強制決定權限 | **系統硬規定** |
| ACL  | Access Control List       | 對某資源列出誰能做什麼  | **資源掛名單** |
| RBAC | Role-Based Access Control | 依角色授權        | **看角色**   |

113-1 還故意放一個假的 **RAC** 當干擾。

**速記一句：**
MAC 看政策，ACL 看名單，RBAC 看角色。

---

## 7. 帳號管理 vs 身分驗證 vs 授權

| 名詞                  | 重點               | 例子            |
| ------------------- | ---------------- | ------------- |
| 帳號管理                | 帳號的建立、停用、盤點、期限管理 | 久未登入帳號、臨時帳號停用 |
| 身分驗證 Authentication | 確認你是誰            | 密碼、MFA、OTP    |
| 授權 Authorization    | 確認你可以做什麼         | 角色權限、最小權限     |

113-2 有題目專門把「密碼複雜度」塞進帳號管理政策裡當陷阱。
113-1 也有把權限管理、未授權存取、驗證措施拆開考。 

**速記一句：**
帳號管理管生命週期，驗證管你是誰，授權管你能做什麼。

---

## 8. MFA vs OTP vs FIDO2

| 名詞    | 重點         | 速記            |
| ----- | ---------- | ------------- |
| MFA   | 多因子驗證      | 不只一種證明身分方式    |
| OTP   | 一次性密碼      | 常作為 MFA 的一種方式 |
| FIDO2 | 無密碼/強式驗證標準 | 偏現代化身分驗證      |

113-1 題目把 MFA、一次性密碼、零信任放在同一組情境裡考。 

**速記一句：**
OTP 是工具，MFA 是做法，FIDO2 是現代標準。

---

## 9. SoD vs RBAC vs 權限審查

| 名詞   | 重點               | 速記      |
| ---- | ---------------- | ------- |
| SoD  | 職務區隔，避免衝突權限落在同一人 | **分人做** |
| RBAC | 用角色來切權限          | **分角色** |
| 權限審查 | 定期檢查權限是否合理       | **定期查** |

113-2 指出有助於 SoD 的措施有職務輪調、RBAC、定期權限審查。

**速記一句：**
SoD 是原則，RBAC 是分權方法，權限審查是持續確認。

---

## 10. 機密性 vs 完整性 vs 可用性

| 名詞                  | 問題核心     | 代表措施       |
| ------------------- | -------- | ---------- |
| 機密性 Confidentiality | 不該看的人看不到 | 加密、存取控制    |
| 完整性 Integrity       | 資料沒被亂改   | 簽章、變更控管    |
| 可用性 Availability    | 需要時可正常用  | 備援、備份、負載平衡 |

113-1、113-2 都有考控制措施與安全屬性的對應。 

**速記一句：**
看不到是機密，沒改掉是完整，用得到是可用。

---

## 11. 防火牆 vs IDS/IPS vs WAF

| 名詞           | 用途          | 速記      |
| ------------ | ----------- | ------- |
| 防火牆 Firewall | 控制網路流量進出    | **守邊界** |
| IDS/IPS      | 偵測/阻擋可疑入侵行為 | **抓攻擊** |
| WAF          | 保護 Web 應用程式 | **護網站** |

113-1、113-2 都常把這幾個放在同一題比較。 

**速記一句：**
防火牆擋流量，IDS/IPS 抓入侵，WAF 顧網站。

---

## 12. VPN vs DMZ vs 網段隔離

| 名詞   | 用途          | 速記        |
| ---- | ----------- | --------- |
| VPN  | 建立安全遠端連線    | **安全通道**  |
| DMZ  | 對外服務區，與內網隔離 | **外網緩衝區** |
| 網段隔離 | 降低橫向移動與部門互通 | **內部分艙**  |

113-1、113-2 都有大量遠端存取、隔離、DMZ 題。 

**速記一句：**
VPN 是連線方式，DMZ 是對外區，隔離是內部切開。

---

## 13. EDR vs Antivirus

| 名詞        | 作用          | 速記          |
| --------- | ----------- | ----------- |
| Antivirus | 偏惡意程式防護     | **防毒**      |
| EDR       | 偵測端點異常行為並回應 | **看行為、能追查** |

113-1 指出 EDR 會記錄活動、分析可疑行為、協助回應，但不是萬能。
113-2 也把 EDR 與防毒、VPN、WAF 放在一起比較。

**速記一句：**
防毒偏擋惡意程式，EDR 偏持續監控與應變。

---

## 14. Zero Trust vs Defense in Depth

| 名詞               | 核心          | 速記         |
| ---------------- | ----------- | ---------- |
| Zero Trust       | 不預設信任，每次都驗證 | **永不預設信任** |
| Defense in Depth | 多層防護、層層阻擋   | **多層防線**   |

113-1 考政府零信任三大核心；113-2 考縱深防禦概念。 

**速記一句：**
零信任重「每次驗」，縱深防禦重「多層擋」。

---

## 15. DDoS vs Credential Stuffing

| 名詞                  | 攻擊目標        | 速記       |
| ------------------- | ----------- | -------- |
| DDoS                | 把服務打掛，影響可用性 | **塞爆服務** |
| Credential Stuffing | 拿外洩帳密去撞登入   | **撞帳密**  |

113-1 題組 3、題組 5 分別考這兩種。 

**速記一句：**
DDoS 打服務，Credential Stuffing 打帳號。

---

## 16. ALE vs ARO vs SLE

| 名詞  | 意思            | 速記       |
| --- | ------------- | -------- |
| SLE | 單一事件發生一次會損失多少 | **一次損失** |
| ARO | 一年大概發生幾次      | **年發生率** |
| ALE | 一年總預期損失       | **年損失**  |

公式：**ALE = SLE × ARO**。

**速記一句：**
ALE 是結果，SLE 是一次，ARO 是頻率。

---

## 17. 風險識別 vs 風險分析 vs 風險評估 vs 風險處理

| 名詞   | 在做什麼        | 速記       |
| ---- | ----------- | -------- |
| 風險識別 | 找出有哪些風險     | **找風險**  |
| 風險分析 | 分析大小、可能性、影響 | **看多大**  |
| 風險評估 | 排序並判斷能否接受   | **排優先**  |
| 風險處理 | 決定怎麼做       | **採取行動** |

113-1、113-2 都有流程題。 

**速記一句：**
先找，再算，再排，再做。

---

## 18. Risk Avoidance vs Reduction vs Transfer/Sharing vs Retention

| 名詞                     | 中文      | 做法         |
| ---------------------- | ------- | ---------- |
| Avoidance              | 風險避免    | 不做、停用、退出   |
| Reduction / Mitigation | 風險降低    | 加控制措施      |
| Transfer / Sharing     | 風險轉移/分擔 | 保險、合約分攤、委外 |
| Retention              | 風險保留    | 接受風險       |

113-1、113-2 都大量考這組。 

**速記一句：**
不做是避免，加控是降低，分給別人是轉移，自己吞是保留。

---

## 19. ISRA vs BIA vs DPIA

| 名詞   | 全名       | 重點        |
| ---- | -------- | --------- |
| ISRA | 資訊安全風險評鑑 | 看資安風險     |
| BIA  | 營運衝擊分析   | 看中斷後對營運影響 |
| DPIA | 資料保護衝擊評鑑 | 看個資處理風險   |

113-1 直接考歐盟場景下 ISRA 與 DPIA 的搭配。

**速記一句：**
ISRA 看資安，BIA 看中斷，DPIA 看個資。

---

## 20. 日誌保存 vs 事件通報

| 名詞   | 重點             | 速記      |
| ---- | -------------- | ------- |
| 日誌保存 | 保留跡證，供追查分析     | **留證據** |
| 事件通報 | 依規定向主管機關或相關方通報 | **報出去** |

113-2 考日誌保存內容；113-1 考事故發生後的通報與處理。 

**速記一句：**
日誌是留，通報是報。

# 最後一頁濃縮速背

## 一眼辨識版

* **27001**：ISMS
* **27701**：PIMS
* **MAC / ACL / RBAC**：存取控制
* **MFA / OTP / FIDO2**：身分驗證
* **Firewall / IDS/IPS / WAF**：邊界與網站防護
* **VPN / DMZ / 網段隔離**：連線與區隔
* **EDR / AV**：端點防護
* **Zero Trust / Defense in Depth**：架構觀念
* **ALE / ARO / SLE**：風險公式
* **Avoid / Reduce / Transfer / Retain**：風險處理
* **ISRA / BIA / DPIA**：不同評鑑

## 最短口訣

* 27001 管資安，27701 管個資
* MAC 看政策，ACL 看名單，RBAC 看角色
* 驗證是你是誰，授權是你能做什麼
* EDR 看行為，WAF 顧網站，VPN 建通道
* 零信任每次驗，縱深防禦多層擋
* ALE = SLE × ARO
* 不做叫避免，加控叫降低
