# 最容易混淆名詞對照表

## 1. 識別 vs 驗證 vs 授權 vs 稽核

| 名詞                | 在問什麼   | 白話意思      | 常見陷阱           |
| ----------------- | ------ | --------- | -------------- |
| 識別 Identification | 你是誰    | 提出身分      | 只有說自己是誰，不代表已證明 |
| 驗證 Authentication | 你真的是你嗎 | 查驗身分真偽    | 有帳號不等於通過驗證     |
| 授權 Authorization  | 你可以做什麼 | 決定可存取哪些資源 | 驗證通過不代表什麼都能做   |
| 稽核 Audit          | 你做過什麼  | 留下可追查紀錄   | 稽核不是授權，也不是驗證   |

**秒記**

* 識別：報名字
* 驗證：看證件
* 授權：決定能進哪個房間
* 稽核：監視器和紀錄簿

---

## 2. 風險辨識 vs 風險分析 vs 風險評估 vs 風險處理

| 名詞   | 核心動作        | 白話意思        | 常見陷阱     |
| ---- | ----------- | ----------- | -------- |
| 風險辨識 | 找出風險        | 有哪些風險       | 還沒開始算高低  |
| 風險分析 | 分析可能性與衝擊    | 風險有多大       | 偏計算與估計   |
| 風險評估 | 拿分析結果判斷優先順序 | 哪些先處理       | 常和分析混用   |
| 風險處理 | 決定怎麼做       | 降低、避免、移轉、接受 | 不是所有都要消滅 |

**秒記**

* 辨識：列清單
* 分析：算大小
* 評估：排優先
* 處理：採行動

---

## 3. 建立全景 vs 風險辨識

| 名詞   | 重點    | 常放進去的內容                  |
| ---- | ----- | ------------------------ |
| 建立全景 | 先理解背景 | 法規、內外部議題、利害關係人、業務情境、技術環境 |
| 風險辨識 | 再找風險  | 哪些資產、弱點、威脅、事件可能造成風險      |

**常見陷阱**
看到「法規要求、主管機關函文、客戶合約」時，常常比較接近 **建立全景** 的背景掌握，不是直接進入處理。

---

## 4. 風險接受準則 vs 風險胃納

| 名詞                 | 白話意思       | 差別           |
| ------------------ | ---------- | ------------ |
| 風險接受準則             | 多大風險算可接受   | 比較像判斷門檻      |
| 風險胃納 Risk Appetite | 組織願意承受多少風險 | 比較像高層容忍方向與程度 |

**秒記**

* 胃納：大方向能吃多辣
* 接受準則：這道菜辣度幾分以下可接受

---

## 5. 固有風險 vs 殘餘風險

| 名詞                 | 白話意思       | 何時看   |
| ------------------ | ---------- | ----- |
| 固有風險 Inherent Risk | 還沒做控制前的風險  | 初始評估  |
| 殘餘風險 Residual Risk | 做完控制後剩下的風險 | 處理後再看 |

**常見陷阱**

* 很多人把「控制後結果」還叫固有風險
* 題目若說已做控制措施，就要想到殘餘風險

---

## 6. 風險避免 vs 風險降低 vs 風險移轉 / 分擔 vs 風險接受

| 名詞        | 白話意思      | 例子            |
| --------- | --------- | ------------- |
| 風險避免      | 不做了       | 結束某業務         |
| 風險降低 / 緩解 | 加控制措施     | 加 WAF、加權限控管   |
| 風險移轉 / 分擔 | 分給別人承擔一部分 | 保險、委外合作       |
| 風險接受      | 知道有風險但接受  | 成本過高，先接受可接受風險 |

**常見陷阱**

* 加保險不是降低，是偏移轉 / 分擔
* 結束業務不是降低，是避免

---

## 7. IAM vs 存取控制 vs RBAC vs SOD

| 名詞   | 重點        | 白話意思         |
| ---- | --------- | ------------ |
| IAM  | 身分與權限整體管理 | 管誰、怎麼登入、能做什麼 |
| 存取控制 | 控制資源使用    | 誰能看、誰能改、誰能執行 |
| RBAC | 角色式授權     | 用角色分權限       |
| SOD  | 職務區隔      | 關鍵工作不要同一人全包  |

**秒記**

* IAM：大傘
* 存取控制：裡面的核心功能
* RBAC：常見授權方法
* SOD：避免內控與舞弊風險

---

## 8. 最小權限 vs 最大特權

| 名詞                   | 意思           | 哪個正確 |
| -------------------- | ------------ | ---- |
| 最小權限 Least Privilege | 只給完成工作所需最少權限 | 正確   |
| 最大特權                 | 給最多最好管理      | 通常錯  |

**常見陷阱**
零信任、IAM、存取控制題很愛用「最大特權」來騙人。

---

## 9. VPN vs WAF vs Firewall vs IDS vs IPS vs EDR

| 名詞       | 主要用途      | 最常見誤解        |
| -------- | --------- | ------------ |
| VPN      | 安全遠端連線    | 不是修補 Web 弱點  |
| WAF      | 保護 Web 應用 | 不是萬用防護       |
| Firewall | 網路邊界流量控管  | 不等於 Web 專用防護 |
| IDS      | 偵測可疑行為    | 偏告警，不一定主動阻擋  |
| IPS      | 偵測並阻擋     | 比 IDS 多了阻擋能力 |
| EDR      | 端點偵測與回應   | 不是邊界設備       |

**秒記**

* 遠端接入：VPN
* Web 防護：WAF
* 邊界管制：Firewall
* 只看不擋：IDS
* 看了會擋：IPS
* 主機端偵防：EDR

---

## 10. 防毒軟體 vs EDR

| 名詞      | 特性           | 差異          |
| ------- | ------------ | ----------- |
| 防毒軟體 AV | 基本惡意程式防護     | 偏簽章、已知威脅    |
| EDR     | 偵測、調查、回應端點威脅 | 更強調行為、調查、反應 |

**常見陷阱**

* 防毒 ≠ EDR
* EDR 通常能力更完整

---

## 11. 備份 vs 備援

| 名詞                       | 白話意思       | 常考方向        |
| ------------------------ | ---------- | ----------- |
| 備份 Backup                | 把資料另存一份    | 復原、RPO、資料恢復 |
| 備援 Redundancy / Failover | 服務不中斷的替代設施 | 可用性、HA、RTO  |

**秒記**

* 備份：找回資料
* 備援：服務不中斷

---

## 12. RTO vs RPO vs MTD vs BIA

| 名詞  | 問的是什麼    | 白話意思         |
| --- | -------- | ------------ |
| RTO | 多久要恢復    | 系統停多久內要回來    |
| RPO | 允許失去多少資料 | 最多能丟多少時間內的資料 |
| MTD | 最長可中斷多久  | 超過就撐不住       |
| BIA | 業務衝擊分析   | 評估哪些系統最重要    |

**秒記**

* RTO：恢復時間
* RPO：資料損失容忍
* MTD：最大可忍中斷
* BIA：先判斷重要性

---

## 13. 可用性 vs 機密性 vs 完整性

| 名詞                  | 在保護什麼    | 常見例子       |
| ------------------- | -------- | ---------- |
| 機密性 Confidentiality | 不該看的人看不到 | 加密、權限控制    |
| 完整性 Integrity       | 資料不能被亂改  | 雜湊、簽章、變更控管 |
| 可用性 Availability    | 要用時能用    | HA、備援、負載平衡 |

**常見陷阱**
資料加密通常偏機密性，不是主要可用性控制。

---

## 14. ISO 27001 vs ISO 27002 vs ISO 27005 vs ISO 27014

| 標準        | 用途           | 常見混淆       |
| --------- | ------------ | ---------- |
| ISO 27001 | ISMS 要求與驗證主體 | 不是單純控制措施清單 |
| ISO 27002 | 控制措施指引       | 不是主驗證標準    |
| ISO 27005 | 資安風險管理       | 不是治理總框架    |
| ISO 27014 | 資安治理         | 不是隱私標準     |

**秒記**

* 27001：制度主體
* 27002：控制指南
* 27005：風險
* 27014：治理

---

## 15. ISO 27701 vs BS 10012 vs IEC 62443 vs ISO 27011

| 標準        | 領域         | 常見考法    |
| --------- | ---------- | ------- |
| ISO 27701 | 隱私資訊管理     | 個資 / 隱私 |
| BS 10012  | 個資保護管理     | 個資制度    |
| IEC 62443 | 工控 / OT 安全 | 製造、工業控制 |
| ISO 27011 | 電信產業資安     | 電信場景    |

**秒記**

* 27701 / BS10012：隱私個資
* 62443：工控
* 27011：電信

---

## 16. 主體 Subject vs 資源 Resource

| 名詞          | 白話意思      | 零信任裡怎麼看     |
| ----------- | --------- | ----------- |
| 主體 Subject  | 發起動作的人或程序 | 使用者、服務、裝置代理 |
| 資源 Resource | 被存取的東西    | 資料、系統、應用、服務 |

**常見陷阱**
題目很愛把所有資料、設備、運算服務都說成主體。很多時候它們更偏 **資源**。

---

## 17. PDP vs PEP vs PE vs PA

| 名詞  | 角色    | 白話意思           |
| --- | ----- | -------------- |
| PDP | 政策決策點 | 做出准不准的總體決策位置   |
| PEP | 政策落實點 | 真正執行攔或放的地方     |
| PE  | 政策引擎  | 判斷該不該放行        |
| PA  | 政策管理者 | 接到判斷後安排執行、建立連線 |

**秒記**

* PE 負責想
* PA 負責安排
* PDP 是決策區
* PEP 是執行口

---

## 18. 資產 Asset vs 弱點 Vulnerability vs 威脅 Threat vs 事件 Event

| 名詞 | 白話意思      | 例子          |
| -- | --------- | ----------- |
| 資產 | 有價值要保護的東西 | 系統、資料、人員、服務 |
| 弱點 | 可被利用的缺口   | 未修補漏洞、弱密碼   |
| 威脅 | 可能利用弱點的來源 | 駭客、惡意員工、天災  |
| 事件 | 實際發生的事情   | 帳號被盜、資料外洩   |

**秒記**

* 資產：被保護
* 弱點：洞
* 威脅：會來打這個洞的人 / 事
* 事件：真的發生了

---

## 19. 個資最小蒐集 vs 目的外利用

| 名詞    | 意思         | 常見陷阱    |
| ----- | ---------- | ------- |
| 最小蒐集  | 只蒐必要資料     | 正確      |
| 目的外利用 | 拿去做原本沒說的用途 | 常違反個資原則 |

**常見錯誤選項特徵**

* 盡可能蒐集全部資料作行銷
* 先收再說，以後可能有用

---

## 20. 委外監督 vs 技術移轉 vs 合約控管

| 名詞   | 核心          | 白話意思           |
| ---- | ----------- | -------------- |
| 委外監督 | 持續看受託者做得好不好 | 稽核、追蹤、驗證       |
| 技術移轉 | 把知識或能力交給委託方 | 教育訓練、文件交付      |
| 合約控管 | 事前把要求寫清楚    | 驗收標準、安全需求、責任分工 |

**常見陷阱**
題目若問「取得前 / 開發前怎麼提升安全」，最優先通常是 **合約控管**，不是等驗收後再補。

---

# 最後濃縮成 12 組必背口訣

1. **識別是誰，驗證真假，授權能做什麼，稽核留紀錄。**
2. **辨識列風險，分析算大小，評估排優先，處理採行動。**
3. **建立全景先看背景，風險辨識才開始找風險。**
4. **固有風險在前，殘餘風險在後。**
5. **避免是不做，降低是加控，移轉是分攤，接受是承擔。**
6. **IAM 是大傘，RBAC 是方法，SOD 是分工。**
7. **最小權限永遠比最大特權更像正解。**
8. **VPN 連遠端，WAF 擋 Web，EDR 看端點。**
9. **備份救資料，備援保不中斷。**
10. **RTO 看恢復時間，RPO 看可丟資料量。**
11. **27001 制度、27002 控制、27005 風險、27014 治理。**
12. **零信任裡，主體是來存取的，資源是被存取的。**
