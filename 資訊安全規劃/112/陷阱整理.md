# 選擇題陷阱整理

## 1. 看到「最適合」「最不適合」「最可能」先別急著選

這科很常不是在問「有沒有關聯」，而是在問 **哪個最貼題**。

### 常見陷阱

* 選項看起來都跟資安有關，但只有一個最符合情境
* 不是問「有沒有幫助」，而是問「是不是最直接、最核心」

### 例子式判斷

* 遠端人員要安全連回公司內網 → **VPN**
* Web 弱點很多 → **WAF** 可能是暫時緩解
* 要追查資料庫異常是否為人為操作 → **資料庫稽核工具**
* 要防外網服務主機被直接打進內網 → **DMZ / 網路區隔**

### 考場提醒

看到題目有：

* 最適合
* 最不適當
* 最可能
* 較無法降低風險

先找「最直接對應情境」那個，不要看到熟悉名詞就選。

---

## 2. 「資安工具都有用」不等於「這題答案是它」

這是最常見失分點。

### 容易誤選的配對

* **WAF**：保護 Web 應用，不是萬用資安解法
* **EDR**：端點偵測與回應，不是網路邊界防護
* **VPN**：安全遠端連線，不是用來修補程式弱點
* **防毒軟體**：基本防護，不代表能解所有進階威脅
* **MSSP**：偏監控與通報，不是直接修補弱點
* **資料庫稽核**：適合追查操作行為，不是用來取代防火牆

### 常見陷阱題型

題目會把幾個都「有點像能用」的工具擺在一起，但只有一個是最貼近需求。

### 秒判斷

* 遠端接入 → VPN
* Web 攻擊 → WAF
* 主機被入侵後的偵測與回應 → EDR
* 追查 DB 誰做了什麼 → DB 稽核
* 邊界過濾 / 區域保護 → 防火牆 / 網段區隔

---

## 3. 風險管理不是背口號，順序常被拿來陰你

很多人知道名詞，但順序一亂就選錯。

### 正確主軸

**建立全景 → 風險辨識 → 風險分析 / 評估 → 風險處理**

### 常見陷阱

把這些故意打亂：

* 還沒建立背景就先分析
* 還沒辨識就先處理
* 把分析、評估、處理混在一起

### 你要記得

* 法規、內外部議題、利害關係人需求，常在 **建立全景**
* 真正列出風險是 **辨識**
* 算可能性、衝擊、排序是 **分析 / 評估**
* 決定加控制、移轉、接受，是 **處理**

---

## 4. 「風險處理」不是把風險降到零

這是超常見陷阱。

### 正確觀念

風險處理目標通常是：

* 降到 **可接受**
* 符合 **風險接受準則**
* 兼顧 **成本效益**

### 常見錯誤選項特徵

* 「必須降到零」
* 「一定要用最高標準」
* 「所有高風險都要完全消除」

這種選項通常很像正確答案，但往往過度絕對。

### 考場口訣

只要看到：

* 一律
* 必須全部
* 完全消除
* 最高標準處理

大多先提高警覺。

---

## 5. 「法規 / 客戶合約 / 主管機關要求」一定常是要納入考量，不要被騙

題目很愛出反向陷阱。

### 正確觀念

下列通常都要納入：

* 法律法規
* 客戶合約要求
* 主管機關公文
* 組織內外部議題

### 常見陷阱句型

* 客戶合約要求「可以不用」納入風險分析
* 法規沒有明講就不用列入
* 只看技術風險就好

這類通常是錯的。

---

## 6. 「重大變更後要不要重評」很愛考

### 正確觀念

遇到重大變更通常要重做或重啟風險評估，例如：

* 上雲
* 新工廠 / 新據點
* 系統大改版
* 業務模式重大改變
* 委外模式改變

### 常見陷阱

* 風險處理完成後就永遠不用再評估
* 已通過驗證就不用再看
* 只要每年一次就夠，不管中間有沒有重大變更

這些都容易錯。

---

## 7. 零信任最容易被混淆成「什麼都不信、什麼都不給」

### 正確觀念

零信任是：

* 永不信任，一律驗證
* 動態依情境決策
* 所有通訊都要保護，不分內外
* 先了解現況，再逐步導入

### 常見陷阱

* 零信任 = 最大特權原則
  錯，應偏最小權限
* 只要通過某個點，就取得最高信任
  錯，零信任強調持續驗證
* 所有資料、設備、服務都是主體
  容易錯，常是把資源和主體混掉
* 一開始先建 PDP / PEP 就對了
  題目若問導入優先工作，通常先盤點現況

### 考場提醒

零信任題最常錯在：

* 主體 / 資源分不清
* PDP / PEP / PE / PA 混淆
* 將「一次驗證」誤當成「持續驗證」

---

## 8. 存取控制題最怕把「識別、驗證、授權、稽核」混在一起

這類題目常用流程名詞混淆。

### 快速分法

* **識別**：你是誰
* **驗證**：你真的是你嗎
* **授權**：你可以做什麼
* **稽核**：你做過什麼，有沒有留紀錄

### 常見陷阱

* 有登入就等於有授權
  錯
* 知道帳號就等於完成驗證
  錯
* 稽核等於存取控制
  不完全對

---

## 9. 離職帳號題幾乎都在考「帳號生命週期管理」

### 常見誤選

看到離職員工還能登入，有人會選：

* 資產處置
* 實體安全
* 已落實存取控制政策

通常最核心還是：

* **使用者存取管理未落實**
* **帳號停用流程失效**

### 考場提醒

只要關鍵字是：

* 離職
* 帳號未停用
* 還能登入
* 權限未收回

先想到 **IAM / 存取控制流程問題**。

---

## 10. 網路架構題最容易被「看起來更強的設備」誤導

### 典型陷阱

題目問的是 ISP 備援，你卻選：

* 多買一台交換器
* 多買一台路由器

但真正單點風險在 **ISP 線路**，答案應是 **備援 ISP**。

### 另一種陷阱

題目問勒索軟體橫向擴散風險，你卻選：

* L3 交換器

但真正重要的是：

* 網路區隔
* VLAN
* AV
* 病毒碼更新

**設備名稱本身不是答案，架構目的才是。**

---

## 11. DMZ 題最容易用「可以連」誤以為「應該放」

### 正確觀念

* 對外服務主機可放 DMZ
* 核心內部管理系統、AD、敏感系統不宜隨便放 DMZ
* 使用者端與伺服器端應避免混在同網段

### 常見陷阱

* AD 放 DMZ 看起來也能連，但風險極高
* User 電腦接到 DMZ switch，看似方便，實際不安全

---

## 12. WAF 題很愛把它講成「萬能防火牆」

### 正確觀念

WAF 最適合：

* HTTP / HTTPS
* Web 弱點的暫時性防護
* 補強 SQLi、XSS 一類風險

### 常見陷阱

* 用 WAF 解決遠端辦公端點風險
* 用 WAF 解決資料庫人為異常
* 用 WAF 解決一般主機惡意程式風險

這些通常不是最佳答案。

---

## 13. 備份題常被誤認為預防控制

### 正確觀念

備份多半偏：

* 復原
* 矯正
* 事故後恢復

### 常見陷阱

* 說備份可預防資料被破壞
* 說備份屬於純偵測措施

如果題目在問控制類型，備份通常偏 **矯正**。

---

## 14. 個資題最容易掉進「蒐集越多越好」陷阱

### 正確觀念

個資 / 隱私常見正解：

* 取得同意
* 最小必要蒐集
* 提供更正 / 修正管道
* 合法、特定目的使用

### 常見錯誤選項

* 儘可能蒐集所有資料方便行銷
* 先蒐再說，以後可能有用
* 不需要讓當事人修正

這種幾乎都錯。

---

## 15. 委外 / 採購題最常考「安全要前移」

### 正確觀念

高分思路是：

* 合約先寫安全要求
* 驗收前要求檢測或證明
* 專責單位參與審查
* SSDLC / 第三方元件控管

### 常見陷阱

* 上線後再補救就好
* 驗收後定期測試，等於前端控制做得夠
* 採用未驗證第三方套件沒關係

---

## 16. OWASP / 安全開發題最容易誤把「重用元件」當萬靈丹

題目提到 Insecure Design 時，常見錯誤是以為：

* 用現成第三方元件就比較安全

其實不是。
重點在：

* 有沒有安全設計
* 有沒有威脅建模
* 有沒有 SSDLC
* 有沒有驗證關鍵流程

**不是外包 / 開源 / 第三方 就自動安全。**

---

## 17. 定量風險計算題最容易多乘一次或少乘一次

### 最常見陷阱

題目給你：

* 資產價值
* 暴露因子
* 年發生率
* 弱點數量

很多人看到「3 個弱點」就多乘 3，但題目如果已說明 **無論弱點多寡，其危害網站 40% 價值**，就不能再乘 3。

### 記法

**ALE = 資產價值 × 暴露因子 × 年發生率**

不是看到數字就全乘。

---

## 18. 「有驗證 / 有通過 ISO」不代表一定沒問題

### 常見陷阱

題目會故意給你：

* 公司已通過 ISO 27001
* 已有認證
* 剛驗證通過

然後問你某缺失怎麼看。

### 正確觀念

* 有認證 ≠ 已完全無缺失
* 仍可能有程序沒落實
* 稽核缺失可能是流程、計算、文件、執行面問題

不要因為看到「已通過驗證」就放鬆警覺。

---

## 19. 「可用性」題不要把加密誤當可用性控制

### 容易誤選

* 負載平衡 → 可用性
* 高可用系統 → 可用性
* 即時備援 → 可用性
* 加密管理系統 → 偏機密性，不是主要可用性考量

題目如果問「哪個不是以可用性為主」，加密很容易是陷阱答案。

---

## 20. 複選題最怕「只會找對的，不會刪錯的」

複選題常有 1 個明顯對，外加 1～2 個半真半假的干擾項。

### 檢查法

每個選項都問自己：

* 是否直接符合題幹
* 是否有過度絕對字眼
* 是否混淆概念
* 是否只是「有點相關」但不是題目要的

# 考前最容易錯的 15 個觀念對照

| 易錯觀念           | 正確觀念          |
| -------------- | ------------- |
| 風險處理要降到零       | 降到可接受即可       |
| 客戶合約可不納入風險分析   | 客戶合約要納入       |
| 通過驗證就表示制度完全沒問題 | 仍可能有缺失        |
| WAF 幾乎什麼都能防    | WAF 主要保護 Web  |
| 備份是預防控制        | 備份多偏矯正 / 復原   |
| 零信任就是全部不給過     | 零信任是持續驗證與動態授權 |
| 驗證過一次就可信任      | 零信任強調持續判斷     |
| 有帳號就算完成授權      | 驗證與授權不同       |
| 離職帳號沒停只是行政疏失   | 這是存取控制重大問題    |
| 多買設備就等於提升安全    | 要看是否對應真正風險點   |
| AD 放 DMZ 沒問題   | AD 放 DMZ 風險高  |
| 蒐集越多個資越方便      | 應最小必要蒐集       |
| 用第三方元件就比較安全    | 仍需驗證與控管       |
| 重大風險處理後不用再看    | 可能仍需再評估       |
| 題目出現數字就全部相乘    | 要看公式與題目條件     |

# 最後給你一份考場秒用口訣

## 看到制度題

先想：**ISO / 法規 / 治理 / 利害關係人**

## 看到風險題

先想：**建立全景 → 辨識 → 分析評估 → 處理**

## 看到存取題

先想：**最小權限 / RBAC / 離職停權 / SOD**

## 看到零信任題

先想：**永不信任，一律驗證 / 動態決策 / 先盤點現況**

## 看到網路題

先想：**VPN / DMZ / 區隔 / 備援 ISP**

## 看到工具題

先想：**這工具是不是正對這個情境，不要看到名詞就選**

## 看到複選題

先想：**每個選項都單獨判斷，不要被一個對的拖著全選**
