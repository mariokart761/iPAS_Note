# 資訊安全規劃實務｜容易混淆名詞對照速記表

## 1. 風險避免 vs 風險降低 vs 風險轉移 vs 風險接受

| 名詞        | 核心意思         | 速記            |
| --------- | ------------ | ------------- |
| 風險避免      | 不做這件高風險活動    | **不做了**       |
| 風險降低 / 修改 | 加控制措施，讓風險變小  | **加控管**       |
| 風險轉移 / 分擔 | 損失責任轉給別人分擔   | **叫別人一起扛**    |
| 風險接受 / 保留 | 風險存在，但在可接受範圍 | **知道有風險，仍接受** |

**常見例子**

* 不自行做金流，交給合格機構：常考 **風險避免**
* 加 MFA、備份、EDR：**風險降低**
* 買資安保險：**風險轉移**
* 低風險項目不再加控管：**風險接受**

---

## 2. 風險識別 vs 風險分析 vs 風險評估 vs 風險處理

| 名詞   | 核心意思            | 速記         |
| ---- | --------------- | ---------- |
| 風險識別 | 找出有什麼風險         | **找風險**    |
| 風險分析 | 分析可能性與影響        | **看多嚴重**   |
| 風險評估 | 跟風險準則比較，決定要不要處理 | **判斷要不要管** |
| 風險處理 | 決定並執行處理方式       | **開始動手管**  |

**速記句**
先 **找出來** → 再 **分析多大** → 再 **評估要不要處理** → 再 **選方式處理**

---

## 3. 風險擁有者 vs 資產擁有者 vs 使用者

| 名詞    | 核心意思              | 速記         |
| ----- | ----------------- | ---------- |
| 風險擁有者 | 有權決定風險怎麼處理、是否接受的人 | **決定風險的人** |
| 資產擁有者 | 對資產負責的人           | **負責資產的人** |
| 使用者   | 使用系統或資產的人         | **用的人**    |

**考點**

* 不是使用者決定風險處理
* 常見正解：**風險擁有者核可風險處理與殘餘風險**

---

## 4. 預防控制 vs 偵測控制 vs 矯正控制

| 類型            | 核心意思 | 例子            | 速記      |
| ------------- | ---- | ------------- | ------- |
| 預防 Preventive | 事前阻止 | MFA、密碼政策、教育訓練 | **先擋住** |
| 偵測 Detective  | 發現異常 | IDS、日誌檢查、監控   | **抓到它** |
| 矯正 Corrective | 事後恢復 | 備份還原、補救措施     | **救回來** |

**超短記法**

* 預防：不讓它發生
* 偵測：知道它發生了
* 矯正：發生後修回來

---

## 5. 機密性 vs 完整性 vs 可用性

| 名詞                  | 核心意思     | 常見關聯                | 速記       |
| ------------------- | -------- | ------------------- | -------- |
| 機密性 Confidentiality | 不該看的人不能看 | 存取控制、MFA、TLS        | **不能外洩** |
| 完整性 Integrity       | 資料不能被亂改  | 防竄改、防 SQL Injection | **不能亂改** |
| 可用性 Availability    | 需要時可以用   | 備援、備份、負載平衡          | **不能掛掉** |

**秒判法**

* 外洩 → 機密性
* 被改 → 完整性
* 不能用 → 可用性

---

## 6. 零信任 vs 傳統邊界防護

| 名詞     | 核心意思          | 速記        |
| ------ | ------------- | --------- |
| 傳統邊界防護 | 比較相信內網，重點守邊界  | **城牆式防守** |
| 零信任    | 內外都不預設信任，持續驗證 | **誰都先不信** |

**考場關鍵**

* 內網不等於安全
* VPN 不等於零信任
* 白名單不等於零信任全部

---

## 7. PEP vs PDP

| 名詞                             | 核心意思      | 速記      |
| ------------------------------ | --------- | ------- |
| PEP (Policy Enforcement Point) | 執行存取決策的點  | **執行者** |
| PDP (Policy Decision Point)    | 判斷准不准存取的點 | **裁決者** |

**速記句**
PEP 會去問 PDP：**這個人能不能進？**

---

## 8. MFA vs 2FA

| 名詞        | 核心意思       | 速記       |
| --------- | ---------- | -------- |
| MFA 多因子驗證 | 兩種以上不同因子驗證 | **多種因子** |
| 2FA 雙因子驗證 | 剛好兩種因子     | **兩種因子** |

**關係**

* **2FA 是 MFA 的一種**
* 題目若問哪個更廣，通常是 **MFA**

---

## 9. Need-to-Know vs Least Privilege

| 名詞                   | 核心意思          | 速記           |
| -------------------- | ------------- | ------------ |
| 僅知原則 Need-to-Know    | 只知道工作需要知道的資訊  | **不該知道就別知道** |
| 最小權限 Least Privilege | 只給完成工作最少需要的權限 | **不該有的權限不給** |

**差異**

* Need-to-Know 偏 **資訊知道多少**
* Least Privilege 偏 **系統能做多少**

---

## 10. SoD vs 管理監督 vs 稽核軌跡

| 名詞       | 核心意思        | 速記       |
| -------- | ----------- | -------- |
| SoD 職務區隔 | 關鍵流程拆開由不同人做 | **分工制衡** |
| 管理監督     | 主管或管理者持續看管  | **有人盯**  |
| 稽核軌跡     | 保留操作紀錄供追查   | **留痕跡**  |

**考點**

* SoD 做不到時，可用管理監督與稽核軌跡補強
* 但補強不等於 SoD 本身

---

## 11. 縱深防禦 vs 單一強力產品防護

| 名詞     | 核心意思        | 速記       |
| ------ | ----------- | -------- |
| 縱深防禦   | 多層控制一起保護    | **多層防線** |
| 單一產品防護 | 靠一套很強的設備或產品 | **只靠一招** |

**考點**

* 縱深防禦幾乎不會等於「只靠一個超強產品」

---

## 12. VPN vs ZTA

| 名詞  | 核心意思         | 速記           |
| --- | ------------ | ------------ |
| VPN | 建立加密通道進入網路   | **安全通道**     |
| ZTA | 對每次存取持續驗證與授權 | **每次都要重新確認** |

**差異**

* VPN 解決的是「通道保護」
* ZTA 解決的是「信任模型與存取決策」

---

## 13. SBOM vs 原始碼檢測 vs 滲透測試

| 名詞    | 核心意思       | 速記         |
| ----- | ---------- | ---------- |
| SBOM  | 列出用了哪些軟體元件 | **成分表**    |
| 原始碼檢測 | 檢查程式碼安全問題  | **看程式碼**   |
| 滲透測試  | 模擬攻擊驗證系統弱點 | **從外面打看看** |

**考點**

* SBOM 不是弱掃工具
* 它是幫你知道「用了哪些第三方元件」

---

## 14. SSDLC vs PSDLC

| 名詞    | 核心意思          | 速記              |
| ----- | ------------- | --------------- |
| SSDLC | 安全系統/軟體開發生命週期 | **把安全放進開發流程**   |
| PSDLC | 產品安全開發生命週期    | **產品導向的安全開發流程** |

**差異**

* SSDLC 比較常泛指系統/軟體
* PSDLC 更偏產品安全治理脈絡

---

## 15. 弱點掃描 vs 滲透測試

| 名詞   | 核心意思         | 速記         |
| ---- | ------------ | ---------- |
| 弱點掃描 | 自動找已知弱點      | **自動找洞**   |
| 滲透測試 | 模擬攻擊驗證可被利用程度 | **真的打進去試** |

**差異**

* 弱掃偏廣、偏自動
* 滲透測試偏深、偏驗證攻擊可行性

---

## 16. 資安事件通報 vs 事件處置

| 名詞   | 核心意思       | 速記        |
| ---- | ---------- | --------- |
| 事件通報 | 依法或依程序回報事件 | **先報出去**  |
| 事件處置 | 控制擴散、補救、復原 | **再處理現場** |

**考點**
題目帶法規時，要記得：

* 不只是救系統
* 還要看是否需通報、分級、評估損害

---

## 17. 備份 vs 備援

| 名詞 | 核心意思          | 速記        |
| -- | ------------- | --------- |
| 備份 | 把資料另外保存，出事後還原 | **留副本**   |
| 備援 | 讓服務不中斷或快速切換   | **不中斷接手** |

**差異**

* 備份偏資料復原
* 備援偏服務持續

---

## 18. EDR vs 傳統防毒

| 名詞   | 核心意思        | 速記           |
| ---- | ----------- | ------------ |
| 傳統防毒 | 偵測已知惡意程式    | **抓已知病毒**    |
| EDR  | 偵測端點異常行為並回應 | **看行為、能追能擋** |

**考點**
勒索軟體題近年常偏好 EDR，不只靠傳統防毒。

---

## 19. 個資法告知義務 vs 放棄權利

| 名詞   | 核心意思             | 速記        |
| ---- | ---------------- | --------- |
| 告知義務 | 告知蒐集目的、對象、方式、期間等 | **要先說清楚** |
| 放棄權利 | 要求當事人放棄法定權利      | **通常違法**  |

**考點**
個資法常考：
可以告知用途，但**不能要求放棄法定權利**。

---

## 20. ISO 27001 vs ISO 27701 vs ISO 42001

| 標準            | 核心主題        | 速記               |
| ------------- | ----------- | ---------------- |
| ISO/IEC 27001 | 資訊安全管理系統    | **ISMS**         |
| ISO/IEC 27701 | 隱私 / 個資管理延伸 | **PIMS / 隱私管理**  |
| ISO 42001     | AI 管理系統     | **AIMS / AI 治理** |

**速記**

* 27001：資安
* 27701：隱私
* 42001：AI

---

## 21. NIST CSF 2.0 vs NIST SP 800-207

| 文件              | 核心主題      | 速記         |
| --------------- | --------- | ---------- |
| NIST CSF 2.0    | 資安框架治理與管理 | **治理框架**   |
| NIST SP 800-207 | 零信任架構     | **ZTA 指南** |

**考點**

* CSF 2.0 新增 **Govern**
* 800-207 是零信任高頻依據

---

## 22. IT vs OT

| 名詞 | 核心意思   | 速記          |
| -- | ------ | ----------- |
| IT | 資訊技術環境 | **辦公資訊系統**  |
| OT | 營運技術環境 | **生產/控制現場** |

**考點**

* OT 要重視可用性與穩定
* IT / OT 通常應區隔
* OT 老舊設備常不支援現代驗證機制

---

## 23. Deepfake 防護 vs 一般社交工程防護

| 名詞          | 核心意思         | 速記         |
| ----------- | ------------ | ---------- |
| 一般社交工程防護    | 防詐騙、釣魚、假冒    | **防騙**     |
| Deepfake 防護 | 防語音/影像高度擬真冒充 | **防超擬真假冒** |

**Deepfake 題常見正解**

* 活體偵測
* 高可信驗證通道
* 多重審批
* SoD

---

## 24. AI Robustness vs Reliability vs Controllability

| 名詞              | 核心意思        | 速記       |
| --------------- | ----------- | -------- |
| Robustness      | 面對干擾仍穩定     | **抗干擾**  |
| Reliability     | 結果穩定可信      | **穩定可靠** |
| Controllability | 可被人管理、限制、介入 | **可控**   |

**速記**

* Robustness：撐得住
* Reliability：做得準
* Controllability：管得住

---

## 25. 通訊加密 vs 資料外洩防護（DLP）

| 名詞   | 核心意思       | 速記       |
| ---- | ---------- | -------- |
| 通訊加密 | 保護傳輸過程不被竊聽 | **路上保護** |
| DLP  | 防止敏感資料不當外流 | **出門要管** |

**差異**

* 加密不等於 DLP
* DLP 也不等於完整個資保護

# 最後一頁｜超短口訣版

### 風險

* 避免：**不做**
* 降低：**加控管**
* 轉移：**叫別人扛**
* 接受：**知道還接受**

### 控制

* 預防：**先擋**
* 偵測：**發現**
* 矯正：**修復**

### CIA

* 機密性：**不能看**
* 完整性：**不能改**
* 可用性：**不能掛**

### 零信任

* **誰都不先信**
* **每次都驗證**
* **最小權限**
* **動態授權**

### SoD

* **不能一人包辦**

### 備份 / 備援

* 備份：**留副本**
* 備援：**不中斷**
